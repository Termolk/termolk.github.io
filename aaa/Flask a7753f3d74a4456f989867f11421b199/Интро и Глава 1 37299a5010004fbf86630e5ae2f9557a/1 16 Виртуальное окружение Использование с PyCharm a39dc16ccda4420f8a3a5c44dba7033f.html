1.16 Виртуальное окружение. Использование с PyCharm

Виртуальное окружение

Самая распространенная IDE PyCharm имеет полную поддержку виртуального окружения.

При создании проекта последняя версия PyCharm сразу добавляет в его корень папку виртуального окружения с именем venv на основе последней доступной версии интерпретатора Python в системе. А вот какую утилиту выберет PyCharm, зависит от версии интерпретатора.

Для Python 3.3 и выше в PyCharm вместо уже встроенной в IDE знакомой нам утилиты virtualenv используется, тоже знакомый нам, встроенный модуль Python venv. Утилита virtualenv в PyCharm используется для более ранних версий, как то: Python 2.x и до версии 3.3. Так как утилиты совместимы, то в качестве имени для обеих утилит virtualenv и venv в диалогах настроек PyCharm используется название Virtualenv.

Если мы хотим изменить поведение PyCharm по умолчанию, то для этого нужно при создании проекта раскрыть подробности пункта **Project Interpreter: New Virtualenv environment [**Интерпретатор проекта: Новое окружение Virtualenv] (1):

Если следовать пункту __New enviroment using__ [Использование нового окружения] (выбран по умолчанию), мы можем создать новое виртуальное окружение. Для этого можно выбрать из списка (Virtualenv, Pipenv и Conda) утилиту работы с виртуальным окружением (2). В большинстве случаев достаточно знакомого Virtualenv.

Доступна также возможность указать путь и имя папки виртуального окружения (3), главное, чтобы она была пуста. По умолчанию папка создаётся в корне проекта и имеет имя venv. И мы помним, что виртуальное окружение может находиться в другом месте, главное тогда задать для его папки понятное имя.

Кроме этого, для создания виртуального окружения можно переиграть сам интерпретатор (4), выбрав из списка или указав его исполняемый файл в операционной системе. Также можно выбрать флажок Inherit global site-packages [Наследовать глобальные site-packages] (5), чтобы добавить в виртуальное окружение уже установленные глобально пакеты, но это, как правило, не требуется.

Если мы решили задействовать ранее созданное виртуальное окружение, то выбираем пункт __Existing interpreter__ [Существующий интерпретатор] и получаем возможность обратиться к списку доступных. Также можно указать виртуальное окружение через выбор в файловой системе (6).

Если мы выбрали интерпретатор из стандартной установки, то виртуальное окружение не будет создано и будет использоваться глобальное пространство пакетов. Для интерпретатора из виртуального окружения будут использоваться пакеты из его установки.

Добавление или изменение виртуального окружения

Мы всегда можем указать или изменить виртуальное окружение для проекта. Это облегчает развертывание проектов, полученных из репозитория, и тестирование приложения в разных окружениях. Для указания или изменения виртуальной среды нужно вызвать диалог Settings / Preferences нажав клавиши Ctrl + Alt + S или выбрав в меню File | Settings для Windows и Linux или PyCharm | Preferences для macOS:

В левой колонке мы выбираем настройки интерпретатора текущего проекта. Далее выбираем через его настройку (1) меню с пунктами добавления или отображения всех настроенных интерпретаторов. В случае добавления открывается окно добавления нового интерпретатора с уже знакомым интерфейсом:

В нём мы можем создать новое виртуальное окружение или выбрать существующее, как мы это уже делали при создании нового проекта.

Под информацией об интерпретаторе находится список доступных пакетов и он соответствует нашему виртуальному окружению (3). Запуск приложения средствами PyCharm также будет выполняться в настроенном для проекта виртуальном окружении. Как видим, PyCharm поддерживает полную интеграцию с виртуальным окружением.
