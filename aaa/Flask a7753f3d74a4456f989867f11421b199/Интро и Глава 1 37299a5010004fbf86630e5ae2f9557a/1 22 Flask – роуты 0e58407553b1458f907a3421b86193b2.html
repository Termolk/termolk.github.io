1.22 Flask – роуты

Мы научимся:

– Выводить разные странички на разные запросы.

– Выполнять разный код для разных случаев.

– Принимать параметры от пользователя (например, запрос в поиске).

– Обрабатывать ситуацию, когда ничего не найдено.

Создание роутов

Как правило, на вашем сайте будет несколько типов адресов, по которым обращаются пользователи – к видео, к плейлисту, к тегу, к главной. И для каждого такого адреса (маршрута, роута) будет функция, которая, например, покажет список видео по конкретному тегу.

Например, когда мы набираем в адресной строке /about/, то хотели бы, чтобы выполнялась функция render_about().

Или когда мы набираем в адресной строке /products/, то хотели бы, чтобы выполнилась функция render_products().

Простейший роут состоит из URL (урла, пути) и функции. Более сложный роут содержит в себе десятки и тысячи разных URL.

А если ввести незарегистрированный ни в одном роуте путь, сервер выдаст ошибку “Not Found”.

Типичные простые роуты

В самом простом случае, одному URL соответствует одна функция.

Предположим, мы хотим вывести список продуктов по адресу /products.

Тогда создадим роут для  /products и объявим для него функцию. Например,

@app.route('/products/')
def render_products():
    return "Здесь будут продукты"

Аналогично сделаем для страницы О проекте:

@app.route('/about/')
def render_about():
	return 'Информация о проекте'

Для главной страницы поменяем название функции и выводимый текст:

@app.route('/')
def render_main():
	return 'Здесь будет главная'

Расширяем наше приложение (скопируйте этот текст):

from flask import Flask # сперва подключим модуль

app = Flask(__name__) # объявим экземпляр фласка

@app.route('/')
def render_main():
	return 'Здесь будет главная'

@app.route('/products/')
def render_products():
	return 'Продукты'

@app.route('/about/')
def render_about():
	return 'Информация о проекте'

app.run() # запустим сервер

@app – что такое декораторы?

Как вы могли заметить, код описания роута начинается с @app.

Возможно, вы уже знаете, что это синтаксис декоратора. При описании роутов он используется для наглядности.

Одна из фишек декоратора, которая делает Flask классным – возможность использовать несколько декораторов для одной функции. Предположим, у нас есть несколько страниц, которые мы пока делаем и мы хотим показывать по ним страничку "еще в разработке". Тогда мы можем сделать так:

@app.route('/profiles/')
@app.route('/about/')
@app.route('/contacts/')
def render_under_construction():
    return render_template('under_construction.html')

Теперь, при обращении на:

/profiles/
/about/
/contacts/

Вы увидите вашу страничку и вам не нужно создавать три функции.

Почитать про декораторы можно здесь:

https://python-scripts.com/decorators

Принимаем один параметр

Бывает так, что нам нужно выводить контент в зависимости от параметра, который пользователь запросил в адресной строке. Например, выводить разные видео для /videos/243/ и videos/455/, и videos/122/.

Тогда нам нужно:

– обозначить, что мы ожидаем параметр id;– передать параметр id в функцию;– использовать параметр в функции.

@app.route('/videos/<video_id>/')
def render_videos_item(video_id):
	return "Здесь будет видео " + video_id

Посмотрим, что получится. Если вы открыли Flask на 0.0.0.0:5000, откройте ссылку:

http://0.0.0.0:5000/videos/mymovie/

http://0.0.0.0:5000/videos/another_movie/

http://0.0.0.0:5000/videos/001/

Принимаем несколько параметров

Иногда в рамках одного роута нам нужно принять несколько параметров из адресной строки. Например, мы делаем книжный магазин и у нас есть страница книги, у которой есть автор и название:

mysite.com/book/johndoe/introduction-to-python/

Или мы находим филиал склада в городе, у него есть город и улица:

mysite.com/stores/ekaterinburg/lenina-ulitsa/

Чтобы принять их, будем использовать такую схему:

@app.route('/book/<author>/<title>/')
def render_book(author,title):
		return "Здесь будет страница книги " + title + " автора " + author

Запустим:

mysite.com/book/johndoe/introduction-to-python/

Получим страницу с надписью:

Здесь будет страница книги introduction-to-python автора johndoe

Встроенное приведение типов <int:uid>

URL – это строка, поэтому  полученные из нее данные передаются в функцию в строковом типе.

В этом легко убедиться, используя type():

  • создайте такой роут и перейдите на /book/1/:
@app.route('/book/<book_id>/')
def render_book(book_id):
    print(type(book_id))
    return ""
  • вы увидите в консоли:
<class 'str'>

Приведение типа к числу – способ 1 (классический)

Обычно, если мы хотим привести строковую переменную к числовому виду, используем int().

Попробуем реализовать это:

@app.route('/book/<book_id>/')
def render_book(book_id):
    book_id = int(book_id)
    print(type(book_id))
    return ""

Попробуйте выполнить этот код и вы увидите в консоли:

<class 'int'>

Все работает, однако у нас появилась лишняя строчка кода и, если мы принимаем 3 параметра, нам нужно обрабатывать каждый. Не очень удобно.

Приведение типа к числу – способ 2 (встроенный, правильный)

Flask (спасибо ему за это) позволяет менять тип переменной "на лету". Вот так:

@app.route('/book/<int:book_id>/')
@app.route('/book/<float:book_id>/')

Попробуйте выполнить этот код:

@app.route('/book/<int:book_id>/')
def render_book(book_id):
    print(type(book_id))
    return ""

И вы увидите в консоли:

<class 'int'>

Аналогично для float.

Полный листинг кода:

from flask import Flask
app = Flask(__name__)


@app.route('/book/<int:book_id>/')
def render_book(book_id):
    print(type(book_id))
    return ""@app.route('/temp/<float:temp_value>/')

def render_temp(temp_value):
    print(type(temp_value))
    return ""


app.run()

Страницы ошибок

Иногда нам хочется сделать страничку 404 красивой.

Вот, как это можно сделать:

@app.errorhandler(404)
def render_not_found(error):
    return "Ничего не нашлось! Вот неудача, отправляйтесь на главную!"

Можно сделать страничку ошибки сервера:

@app.errorhandler(500)
def render_server_error(error):
    return "Что-то не так, но мы все починим"

Таким образом можно создавать веселые и нестандартные страницы ошибок.

Код ответа сервера

Чтобы поисковые системы правильно понимали страницы ошибок, нужно обязательно указать код ответа сервера. Когда мы выполним код выше, страничка "что-то не так..." будет отдана сервером с кодом 200, что можно увидеть во вкладке Network браузера.

Чтобы код ответа сервера был корректным,  допишем eго к return:

@app.errorhandler(500)
def render_server_error(error):
    return "Что-то не так, но мы все починим", 500

Параметр е

Объект e можно использовать для вывода ошибки или логирования.

Такой код:

@app.errorhandler(404)
def render_server_error(error):
    return "Что-то не так, но мы все починим:\n{}".format(error), 404

Выведет:

Что-то не так, но мы все починим:
404 Not Found: The requested URL was not found on the server. If you entered the URL manually please check your spelling and try again.
