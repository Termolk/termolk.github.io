1.9 HTTP-сообщения

HTTP-сообщения: Методы и статусы

Протокол HTTP основан на архитектуре «клиент-сервер», построенной на основе обмена сообщениями. HTTP-сообщения — это способ обмена данными между клиентами и серверами в Интернете. Есть два типа сообщений: запросы и ответы.

Запрос — это операция, которую клиент хочет выполнить на сервере, а ответ — это ответ сервера на входящий запрос. Обычно программистам не нужно беспокоиться о создании HTTP-сообщений, поскольку они выполняются браузерами, приложениями и веб-серверами.

Формат сообщений

В протоколе HTTP все сообщения состоят из текстовых строк. И запросы, и ответы имеют примерно одинаковый стандартизированный формат.

1. Стартовая строка, которая может варьироваться:

— для запросов указывается тип запроса (метод) и URL-адрес, куда его отправить (цель);     — для ответов он содержит код состояния для определения успешности операции.

2. Заголовки, которые описывают сообщение и передают различные параметры.3. Тело, в котором находятся данные сообщения.

Стартовая строка и заголовок являются обязательными атрибутами, остальные части могут быть пустыми.

Полный формат может быть довольно сложным для начинающих, поэтому мы рассмотрели только его часть, которая является наиболее важной для понимания общих принципов.

Упрощенное HTTP-взаимодействие

Вот упрощенное HTTP-взаимодействие между клиентом браузера и сервером. Клиент и сервер взаимодействуют посредством запросов и ответов, которые следуют изученному формату.

Обратите внимание, что существуют другие возможные типы клиентских программ, а не только браузер. Вы даже можете написать свой собственный HTTP-клиент и взаимодействовать с серверами. Единственное требование — такая программа всегда должна следовать формату сообщения.

Методы

HTTP определяет набор методов запроса, которые определяют, какое действие будет выполнено для данного ресурса. Несмотря на то, что их имена могут быть существительными, эти методы запроса иногда называют HTTP-метод.

Давайте посмотрим на наиболее часто используемые методы запросов:

  • GET используется только для получения данных с сервера;
  • POST используется для отправки данных на сервер;
  • HEAD запрашивает данные с сервера так же, как метод GET, но без тела ответа.

Каждый раз, когда вы нажимаете на ссылку, вы в основном сообщаете браузеру, что хотите получить эту страницу. Когда вы хотите войти на свой любимый сайт, вы отправляете свой логин и пароль, чтобы получить доступ к нему.

Есть больше доступных методов для изучения. Вам не нужно запоминать их все прямо сейчас.

Коды состояния

Для нормальной работы компьютерных программ и веб-страниц, которые работают через HTTP, за исключением содержимого страницы, сервер возвращает трехзначный код, который указывает ответ запроса. С помощью этого кода можно перенаправить клиента на другой сайт или указать изменение страницы, а также обнаружить ошибку в обработке данных.

В настоящее время стандарты определяют пять классов кодов состояния:

1xx: InformationalКоды, начинающиеся с «1», называются информационными кодами. Они сообщают о том, как обрабатываются клиентские запросы.
2xx: SuccessДанный код сообщает, что запрошенное клиентом действие успешно принято к обработке.
3xx: RedirectionЭто означает, что для выполнения запроса необходимо предпринять дальнейшие действия.
4xx: Client ErrorОн сообщает об ошибках на стороне клиента.
5xx: Server ErrorКод указывает, что операция была неудачной из-за ошибки сервера.

Браузеры отображают сообщения об ошибках, чтобы пользователи понимали, что что-то пошло не так, вместо того, чтобы продолжать смотреть на пустую страницу в ожидании загрузки контента.
