1.10 PyCharm и установка Python

Какую среду программирования (IDE, integrated development environment) лучше использовать, когда вы только начинаете?

Мы рекомендуем PyCharm, разработанную JetBrains специально для Python: она мощная, с удобным интерфейсом, умеет автодополнять код, индексировать проекты, давать советы по стилю программирования, управлять пакетами и еще сотни вещей, которые вы оцените в будущем. PyCharm доступен в бесплатной версии "Community" (ее достаточно для курса) и профессиональной (платной) "Professional".

Узнайте больше о PyCharm, посетив официальный сайт.

Найдите руководство и установите PyCharm здесь.

Если вы еще не установили Python – скачайте и установите его здесь: https://www.python.org/downloads/ .

Первый проект в PyCharm

Теперь мы напишем ваш первый скрипт «Hello, World» с использованием PyCharm (мы предполагаем, что вы уже установили его на свой компьютер).

2. После этого вам предлагается установить несколько плагинов. В данный момент они вам не нужны, но зато теперь вы знаете, что PyCharm можно настроить по своему желанию практически для всего. Как видите, вы всегда можете установить новые плагины в Settings | Plugins.

3. После этих настроек у нас наконец-то появилось действительно важное окно — создание или открытие нового проекта. Доступны 3 варианта: вы можете либо создать новый проект, либо открыть существующий, либо воспользоваться любой системой контроля версий, такой как Git или Bitbucket. Поскольку вы только начинаете программировать, у вас, вероятно, нет существующих проектов, так что давайте выберем «Создать новый проект».

4. Далее вам нужно выбрать путь к вашему будущему проекту, где будут храниться все принадлежащие ему файлы. Таким образом, вы даете имя своему проекту. В моем случае название проекта будет «FirstProject». Но есть еще и раскрывающееся меню Project Interpreter: New Virtualenv environment. Это означает, что ваш проект будет создан с новыми настройками по умолчанию. В основном это означает, что он будет изолирован от других проектов. Давайте рассмотрим этот момент более подробно.

Представьте, что у вас есть 3 проекта, каждый из которых написан на отдельной версии Python: 2.7, 3.5 и 3.7. На них также могут быть установлены разные наборы библиотек. Чтобы избежать конфликтов и сделать процесс разработки более понятным, мы создаем все проекты в разных папках с разными интерпретаторами. На самом деле, пока достаточно просто нажать кнопку «Создать», но давайте взглянем на настройки интерпретатора.

5. Первое поле — это путь к новой виртуальной среде. Вы можете думать о виртуальной среде как об изолированном «псевдо» дистрибутиве Python с его собственными функциями: доступными библиотеками, версией Python и pip — встроенным менеджером пакетов Python, который позволяет устанавливать внешние библиотеки. Base interpreter — это путь к настоящей версии Python, которую вы загружаете и устанавливаете на свой компьютер.

Допустим, вы загрузили Python 2.7 и установили его в C:/python2.7, а Python 3.7 — C:/python3.7.

Теперь, если вы хотите создать новый проект на Python 2.7, вы выберете C:/python2.7/python.exe в качестве базового интерпретатора для этого проекта. Если вы хотите начать новый проект на Python 3.7, вы выберете C:/python3.7/python.exe в качестве базового интерпретатора проекта, написанного на Python 3.7.

Каждый из этих проектов будет иметь свои зависимости — библиотеки, доступные и необходимые для конкретного проекта.

6. Если вы откроете корневой каталог, то увидите, что есть еще одна папка — ваша новая виртуальная среда, которая содержит все настройки Python для данного конкретного проекта. Как мы уже знаем, настройки — это версии Python, pip и библиотеки.
