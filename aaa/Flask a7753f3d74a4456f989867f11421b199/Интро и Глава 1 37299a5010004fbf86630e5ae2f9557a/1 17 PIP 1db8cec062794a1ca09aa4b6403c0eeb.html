1.17 PIP

Огромное количество людей по всему миру используют Python для решения самых разных задач. По сути, это означает, что в открытом доступе уже есть множество решений для весьма большого числа проблем. Это может быть особенно полезно, когда вы работаете над своими собственными проектами, поскольку, скорее всего, вы сможете найти подходящий пакет для конкретной задачи и эффективно использовать его для ее решения. Теперь время узнать немного о стандартных инструментах управления пакетами в Python.

Что такое PIP?

Вы, вероятно, уже знакомы со стандартной библиотекой Python, она содержит огромное количество полезных встроенных модулей и устанавливается вместе с вашим дистрибутивом Python. Фактически, еще одна вещь, которая также идет с дистрибутивом Python (начиная с Python 3.4), это стандартный менеджер пакетов, называемый pip (обычно расшифровывается как «Pip Installs Packages»).

Pip предназначен как для расширения функциональности стандартной библиотеки путем установки дополнительных пакетов на ваш компьютер, так и для того, чтобы вы могли делиться своими собственными проектами и тем самым способствовать развитию Python.

Теперь давайте удостоверимся, что у вас установлен pip. Все, что вам нужно сделать, это открыть командную строку / терминал и запустить эту строку:

pip --version

Вывод должен сообщить вашу текущую версию pip. Например, последняя версия на 1 апреля 2020:

pip 20.0.2

Если он не установлен (или вы хотите обновить его), пожалуйста, следуйте инструкциям по установке специально для вашей операционной системы.

Возможности PIP

Поскольку pip рассчитан в первую очередь для установки пакетов, наиболее очевидной и важной командой для начала будет install. Посмотрите на следующую строку:

pip install some_package

Установка действительно очень проста. Однако, если вас интересует определенная версия пакета, вам нужно указать ее после имени пакета следующим образом:

pip install some_package==1.1.2

Или, по крайней мере, определить минимальную версию:

pip install "some_package>=1.1.2"

Обратите внимание, что последнее выражение должно быть заключено в двойные кавычки, чтобы оператор сравнения интерпретировался без проблем.

Еще одна полезная вещь — команда show . Она показывает информацию об установленных пакетах, например, их версию, автора, лицензию, местоположение или требования. Вот обобщенный пример:

pip show some_package

Также полезно знать команду list — она перечисляет все пакеты, которые вы установили на свой компьютер в алфавитном порядке:

pip list

Если вы напечатаете команду list с опцией — outdated или просто -o , вы получите список устаревших пакетов в сочетании с текущей и последней доступной версией.

pip list --outdated

или в несколько более короткой версии:

pip list -o

После выполнения одной из упомянутых строк вывод будет такой:

first_package (Current: 2.1.1 Latest: 3.0.1)
second_package (Current: 4.2.1 Latest: 4.2.2)

Чтобы удалить пакет с вашего компьютера, выполните команду удаления:

pip uninstall some_package

При разработке вашего проекта может быть полезно сохранить список пакетов для установки  в специальном файле (см.  Requirements File Format). Это удобно, потому что вы можете устанавливать пакеты прямо из него:

pip install -r requirements.txt

Конечно, вы не должны сами писать файл со списком всех названий необходимых пакетов. Для получения файла с требованиями достаточно запустить приведенный ниже код (символ ">" означает, что мы перенаправляем поток вывода командной строки в файл)

pip freeze > requirements.txt

Давайте рассмотрим строку выше в деталях. freeze — команда, используемая для получения всех установленных пакетов в формате требований. Таким образом, все пакеты, которые вы установили перед выполнением команды и предположительно использовали в каком-либо проекте, будут перечислены в файле с именем «requirements.txt». Кроме того, будут указаны их точные версии (см. Requirement Specifiers).

Рассмотрим пример вывода команды freeze:

beautifulsoup4==4.7.1
nltk==3.4.1
numpy==1.16.3
scikit-learn==0.21.1
scipy==1.3.0

Важно понимать, что команда freeze на самом деле касается всех установленных библиотек и может создать очень длинный список, если вы не используете виртуальное окружение.По этой и другим причинам, мы рекомендуем использовать более осознанный подход и самостоятельно пересматривать полученный файл!

Подводя итоги

В целом, мы узнали основные моменты работы с pip:

  • как установить пакеты (либо конкретную версию, либо неспецифическую),
  • как создать файл требований и использовать его для установки,
  • как получить информацию об установленных пакетах,
  • и, наконец, как удалить пакеты.

Для получения более подробной информации смотрите документацию или запустите команду help.
