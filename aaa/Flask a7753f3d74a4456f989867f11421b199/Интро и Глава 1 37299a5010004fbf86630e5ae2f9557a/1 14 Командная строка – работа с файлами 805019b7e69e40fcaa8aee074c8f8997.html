1.14 Командная строка – работа с файлами

Вы уже знаете, как выполнять некоторые базовые действия с каталогами, например, переходить от одного каталога к другому или создавать/удалять каталоги через графический интерфейс. Вы также можете сделать это через командную строку. В этом разделе мы рассмотрим этот процесс подробно.

Мы рассмотрим довольно много примеров. Обратите внимание, что в этих примерах  $ символом является приглашение командной строки — специальный символ, указывающий, что компьютер готов принимать команды. В Windows он обычно выглядит следующим образом:  >.

Распечатать рабочий каталог

В UNIX-подобных системах  pwd ( Print Working Directory ) команда печатает информацию о каталоге, в котором вы находитесь в настоящее время. Например:

$ pwd
/Users/username

В Windows это делается с помощью  cd команды без аргументов, хотя особой необходимости в такой команде нет.

C:\Users>cd
C:\Users

Попробуйте запустить ее: откройте интерфейс командной строки, введите pwd или  cd , а затем нажмите Enter. Он напечатает ваш  домашний каталог — тот, с которого вы начинаете, когда открываете новое окно CLI. Обычно это выглядит примерно так: Macintosh HD/Users/username  или  C:\Users\username.

Ну что, прогуляемся? Давайте попробуем пройтись от одного каталога к другому.

Смена каталога

Команда cd (Change Directory/Изменить каталог) (доступна как в Windows, так и в UNIX-подобных системах) позволяет вам изменить каталог, в котором вы находитесь:

UNIX:

$ cd Documents
$ pwd
/Users/username/Documents

Windows:

C:\Users>cd C:\Program Files

C:\Program Files>
  • Чтобы быстро вернуться в свой домашний каталог, используйте cd без аргументов или cd ~в UNIX-подобных системах, и cd \ в Windows.
  • Чтобы вернуться в предыдущий каталог в UNIX-подобной системе используйте cd - и cd .. для возврата в родительский каталог (каталог на один уровень выше текущего каталога; в нашем примере username является родительским каталогом для Documents, а Users является родительским каталогом для username).
  • С помощью cd команды вы можете пройтись по всей директории. Для разделения разных каталогов на вашем пути используйте косую черту. Например, команда cd Documents/my_dir переводит вас в Documents,  а затем сразу в my_dir.

Не забывайте, что UNIX-подобные системы используют косую черту —  /, а Windows  бэкслэш — \ !

Список содержимого

ls на UNIX или dir на Windows, печатает текущее содержимое каталога:

$ ls
stuff old_files text.txt photo_1.jpg photo_2.jpg

Создание и удаление файлов и каталогов

Чтобы создать новый пустой каталог, используйте команду mkdir ( Make Directory ), указав имя нового каталога после команды:

$ mkdir new_dir
$ ls
stuff old_files new_dir text.txt photo_1.jpg photo_2.jpg

Чтобы безвозвратно удалить файлы или каталоги в UNIX-подобных системах, используйте команды rm ( Remove / Удалить ) и  rmdir ( Remove Directory / Удалить каталог) соответственно. У Windows есть команда del, которая удаляет все.

$ rmdir new_dir
$ ls
stuff old_files text.txt photo_1.jpg photo_2.jpg
$ rm photo_1.jpg photo_2.jpg
$ ls
stuff old_files text.txt

Команды mkdir и rmdir работают как в UNIX-подобных системах, так и на Windows.

Копирование и перемещение

Чтобы скопировать или переместить файлы или каталоги, используйте команды cp и mv  ( копирование и перемещение  в Windows). Они принимают два или более имен файлов или каталогов в качестве аргументов. Последний аргумент считается местом назначения копирования или перемещения, а все аргументы перед ним считаются источниками. Другими словами, все файлы и каталоги до последнего копируются или перемещаются в последний каталог.

сp и copy также можно использовать для копирования одного (но только одного!) Файла в другой, например:

cp source_file.txt target_file.txt

В Windows copy, используемая только с одним именем файла в качестве аргумента, напечатает содержимое этого файла.

Чтение файлов

Теперь вы можете легко перемещаться по файловой системе через интерфейс командной строки. Но как насчет файлов? Можем ли мы читать файлы или записывать их из командной строки?

Чтобы прочитать содержимое файла, используйте  команду cat(что странно расшифровывается как  conCATenate ) в UNIX-подобных системах и type в Windows. Также для Windows вызов copy с одним именем файла в качестве аргумента также приведет к печати содержимого файла.

$ cat text.txt
Twas brillig, and the slithy toves
Did gyre and gimble in the wabe:
All mimsy were the borogoves,
And the mome raths outgrabe.

Создание новых файлов

В UNIX-подобных системах есть две команды, позволяющие создавать один или несколько файлов:

  • touch просто создает пустой файл или файлы с заданным именем
  • mkfile (Make File) создает один или несколько файлов заданного размера (в байтах), заполненными нулями; например, mkfile 8 new_file.txt создаст 8-байтовый файл с именем new_file.txt.

В Windows может также использоваться echo > filename ; однако он позволяет создавать только один файл за раз. Обратите внимание, что в Windows PowerShell вам нужно нажать F6 + Enter после этого, чтобы выйти.

Предупреждение: не забудьте указать расширение при создании файла с помощью командной строки, иначе у него не будет расширения!

В отличие от большинства графических текстовых редакторов, которые позволяют пользователям записывать что-либо в файл перед его сохранением, эти утилиты создают файл и немедленно сохраняют его, не позволяя пользователю изменять его содержимое.

Редактирование файлов

Существует способ записи в файл через командную строку. Простая команда с именем echo (доступная как в UNIX, так и в Windows) может использоваться для записи нового содержимого в конец файла (echo "некоторый новый контент" >> <имя файла>) или для полного переопределения существующего файла новым содержимым (echo "некоторый новый контент" >  <имя файла> ). Эти варианты лучше, чем ничего, но все еще не очень полезны. Есть ли способ отредактировать файл через командную строку таким же образом или лучше просто отредактировать его в Google Docs?

На самом деле, есть. Все операционные системы имеют встроенные консольные текстовые редакторы, которые позволяют записывать в файл из командной строки. Как в Windows, так и в UNIX-подобных системах Vim доступен с помощью команды vim(или vi ), а Emacs — с  помощью команды emacs. UNIX-подобные системы также имеют Nano, доступ к которому можно получить с помощью команды nano . Чтобы глубже понять эти текстовые редакторы, не стесняйтесь посмотреть FAQ и документы для Vim, Emacs и Nano.

Мы не будем вас обманывать: они довольно сложные, и о каждом написаны целые книги.

Консольные текстовые редакторы также сильно отличаются от текстовых редакторов с графическим интерфейсом и даже могут показаться нелогичными, поскольку они берут свое начало со времен, когда графические интерфейсы не существовали вообще или были редкостью.

Но подробности мы любезно оставим вам на добровольное изучение, если вы хотите докопаться до сути.
