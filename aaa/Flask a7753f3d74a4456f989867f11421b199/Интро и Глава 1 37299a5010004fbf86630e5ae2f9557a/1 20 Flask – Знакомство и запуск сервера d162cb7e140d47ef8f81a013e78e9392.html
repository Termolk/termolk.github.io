Flask – Знакомство и запуск сервера

Мы изучили основы клиент-серверной архитектуры, протокола HTTP и узнали про теги HTML.

Flask использует все эти технологии: Ваше Flask приложение запускается на сервере. Возвращает на запрос клиента HTML. Этот HTML — результат выполнения кода, который вы написали.

В этом уроке мы познакомимся с принципами работы сервера, узнаем о фреймворках и напишем первые строчки кода для работы с микрофреймворком Flask, на котором будет построено наше приложение.

Логика работы веб-сервера

С помощью языка Python мы умеем генерировать строки, а с помощью простого HTTP сервера умеем передавать содержимое текстового или html файла. Прежде, чем переходить от этого к созданию динамических веб сайтов, определимся, что мы от них хотим.Типичный пример такого сайта – интернет-магазин.Рассмотрим, как он должен работать:

1. Пользователь запрашивает site.com/фляжки, это GET запрос на опредленный порт, например, 80.
2. Сервер замечает обращение на порт, передает запрос скрипту Python, например app.py который этот порт слушает.
3. app.py получает из адресной строки параметр "фляжки" и обращается к базе данных за списком фляжек.
4. Получив данные, скрипт обрабатывает их, например, добавляет скидку, проверяет наличие итд.
5. Когда все готово, скрипт на основе данных строит HTML документ, создавая длинную строку.
6. Эту строку сервер отдает клиенту в формате html, в качестве ответа на запрос.
7. Браузер пользователя отрисовывает страничку.

Допустим теперь, что пользователь хочет найти определенную модель фляжки, например "турист":

1. Пользователь запрашивает /site.com/search?s=фляжка_турист.
2. Сервер замечает обращение на порт, передает запрос скрипту Python, например app.py который этот порт слушает.
3. app.py получает из адресной строки параметр "s" – строка поиска и ищет по базе данных все подходящее.
4. Найдя товары он обрабатывает их.
5. Когда все готово, скрипт на основе данных строит HTML документ, создавая длинную строку.
6. Эту строку сервер отдает клиенту в формате html, в качестве ответа на запрос.
7. Браузер пользователя отрисовывает страничку.

Допустим теперь, что мы работаем над другим проектом и занимаемся сервисом для поиска попутчиков.

Пользователи приходят к нам и хотят найти, кто когда и куда поедет. Логика будет такой:

1. Пользователь запрашивает /site.com/search?from=Москва&to=Владимир&when=today.
2. Сервер замечает обращение на порт, передает запрос скрипту Python, например app.py который этот порт слушает.
3. app.py получает из адресной строки параметры "from"  "to" "when"  и создает запрос базе данных.
4. База данных возвращает подходящие поездки, скрипт сортирует их по релевантности.
5. Когда все готово, скрипт на основе данных строит HTML документ, создавая длинную строку.
6. Эту строку сервер отдает клиенту в формате html, в качестве ответа на запрос.
7. Браузер пользователя отрисовывает страничку.

Как можно заметить, общий принцип для разных проектов достаточно схож:

– Сервер слушает порт и передает данные скрипту на Python.

– Скрипт на Python смотрит, что пользователь запросил в адресной строке.

– Скрипт на Python выполняет разные алгоритмы для разных запросов.

– Скрипт на Python формирует HTML строку.

– Сервер передает HTML, CSS, JS, графические файлы пользователю.

– Браузер пользователя отрисовывает страничку.

Как и всегда в таких случаях, типичные операции можно автоматизировать, сформировав заготовки и готовые инструменты. Логично предположить, что создаются готовые решения и приемы их использования.

Фреймворки и микрофреймворки

Программы можно сравнить со снежинками: они одновременно уникальны и схожи одна с другой. Такие сходства могут быть использованы в интересах разработчика. В программировании обычной практикой является повторное использование кода, упакованного в библиотеки, чтобы упростить разработку. Такие библиотеки существуют для большинства языков программирования; они предоставляют хорошую документацию и надежный код, который могут использовать повторно сотни и тысячи разработчиков.

Крупные приложения, такие как интернет-магазины, онлайн-банки или социальные сети, часто нуждаются в одних и тех же типичных компонентах и функциях, таких как авторизация пользователей, взаимодействие с базой данных, отправка уведомлений и так далее. Чтобы использовать их повторно, разработчики создают специальный вид программного обеспечения, называемый фреймворками.

Фреймворк (от framework «остов, каркас, структура») — это универсальное программное обеспечение многократного использования, которое облегчает разработку типовых приложений или их частей. Он состоит из шаблонов структурированного кода и предоставляет общие функциональные возможности, которые могут быть легко расширены для нужд конкретного приложения.

Flask называют микрофреймворком.

Приставка "микро-" означает, что таких модулей немного, в них можно разобраться быстро и фреймворк не содержит сотни модулей на все случаи жизни. В больших фреймворках, например, Django, есть модули для работы с базой данных, формами, пользователями, администрированием и правами доступа. Во Flask есть сервер, роуты, шаблонизация,  работа с сессиями и авторизацией.

Поскольку это микрофреймфорк, потребуются минимальные усилия, чтобы разобраться и запустить простой сервис.

Кто разрабатывает фреймворки?

Создатели фреймворков – программисты, которые придумывают новый способ организовать архитектуру приложения и готовы вкладывать силы и время, чтобы поделиться им. Фреймворки часто появляются, как результат изобретений при работе над большим проектом. Например, фреймворк Ruby on Rails был создан David Heinemeier Hansson на основе его работы в компании 37signals.

Популярный и мощный Python фреймворк Django был создан осенью 2003 года, когда веб-программисты из газеты Lawrence Journal-World, Adrian Holovaty and Simon Willison, начали использовать Python для создания приложений. https://ru.wikipedia.org/wiki/Django .

Крупные компании также иногда выпускают и поддерживают фреймворки. В качестве двух примеров можно назвать Angular от Google https://ru.wikipedia.org/wiki/AngularJS и React от Facebook https://ru.wikipedia.org/wiki/React. А вот фреймворк Vue.js создан независимой командой.

А бывает и так, что компания, разработавшая новый язык программирования, выпускает к нему фреймворк. Например, фреймворк Ktor для Kotlin разработан командой из JetBrains https://ktor.io/index.html .

Автор фреймворка Flask – Armin Ronacher. По словам автора, изначально идея создания такого фреймворка была первоапрельской шуткой. Сейчас Flask – один из самых популярных фреймворков для Python.

Пора научиться работать с Flask и написать первое приложение!

Знакомство с Flask и запуск сервера

Поскольку Flask не часть стандартного Python, нам потребуется его установить.

Это можно сделать с помощью PIP:

pip install Flask

или

pip3 install Flask

Начинаем писать код

Чтобы познакомиться с Flask, cоздадим приложение которое будет при обращении к корню сайта (site.com/ или localhost/) возвращать 'Hello, World!' и протестируем его! Для этого:

  1. Создадим папку и перетащим ее в Pycharm или создадим новый проект в Pycharm.
  1. Создадим в корне файл app.py (или main.py, название не важно).
  1. Скопируем код приложения, которое подключает фласк, выполняет функцию "hello" при обращении к роуту "/" и слушает адрес 0.0.0.0, порт 8000 локального компьютера.
  1. Запустим наш Python файл app.py .
  1. Обратимся к нужному порту локального хоста из браузера.

Итак, создайте папку, в ней файл app.py, в него скопируйте код и запустите!

from flask import Flask    # сперва подключим модуль

app = Flask(__name__)      # объявим экземпляр фласка

@app.route('/')            # объявим путь /def hello():               # объявим функцию для пути /return 'Hello, World!'   # выведем текст, который будет при обращении на /

app.run('0.0.0.0',8000)    # запустим сервер на 8000 порту!

Если все хорошо, вы увидите сообщение:

Flask запустится и начнет работать.

Перейдите в браузере по ссылке, чтобы увидеть результат (для пользователей UNIX-систем):

http://0.0.0.0:8000/

Для пользователей Windows в адресной строке браузера надо указать:

http://localhost:8000

Про ошибки в коде

Если вы допустили ошибку в коде, сервер не запустится. Например, рука дрогнула и мы стерли двоеточие при объявлении функции. Мы получим сообщение SyntaxError: invalid syntax .

Это обычные ошибки Python и вы знаете, что с ними делать.
