1.24 Jinja – Основы шаблонизации + переменные

Jinja – Основы шаблонизации

Если вы использовали html-строки внутри Python, то замечали: читать код, где перемешаны два языка не очень удобно, особенно, когда html становится огромным. Шаблонизаторы позволяют нам хранить html-шаблоны отдельно от логики и данных, и выводить в нужных местах переменные, списки и словари.

Если на сайте есть статья с заголовком, текстом и автором, шаблонизатор примет эти переменные и сгенерит страничку со статьей. Если у вас на сайте есть меню, шаблонизатор примет список словарей и создаст на его основе нужные ссылки. Если на сайте есть список тегов, шаблонизатор примет список и создаст на его основе теговое облако. Вот так:

Обычно шаблонизаторы умеют:

– выводить значения переменных в html-шаблоне,

– форматировать значения (округлять, делать большие буквы и.т.д.),

– реализовывать условия внутри html-шаблона,

– реализовывать циклы внутри html-шаблона,

– выводить результат запуска стандартных и пользовательских функций в html-шаблоне.

В этом топике вы научитесь:

– подключать Jinja,

– создавать простые шаблоны,

– передавать переменные в шаблонизатор,

– использовать Jinjа вместе с Flask.

Что мы знаем о генерации строк

Если вам нужно добавить вычисленные значения в html код на python, у вас есть несколько вариантов это сделать.

1. Первый способ – слияние строк:

name = "Steven"
print ("<h1>Hello "+name+"</h1>" )

# >> Hello Steven

2. Второй способ – форматирование строк:

name = "Steven""<h1>Hello {}</h1>".format(name)

# >> Hello Steven

3. Третий способ – F-строки:

name = "Steven"
print ( f"<h1>Hello {name}</h1>" )

# >> Hello Steven

Однако, даже при использовании форматирования, если вы создаете HTML строку для передачи пользователю, основной файл проекта будет загроможден тегами, а если в коде встречаются условия или циклы, логика и строки для вывода будут так перемешаны, что их неудобно будет читать и редактировать. Радикальное решение – разделить логику (функции и операторы python) и представление (теги) по разным файлам. Для этого мы сначала вычисляем и готовим все переменные, списки и словари, затем передаем их в шаблон, где шаблонизатор ставит их в специальные места и собирает (рендерит) готовый html документ.

Именно этим и занимаются шаблонизаторы, вроде Jinja. Давайте познакомимся с ним!

Использование файлов шаблонов

Давайте научимся хранить шаблоны в отдельных html файлах и загружать их при необходимости.

По умолчанию, Flask будет искать шаблоны в папке "templates"

– Создадим в нашей папке проекта папку templates.– Создадим в папке templates файл test.html.– Создадим в test.html простой шаблон с текстом.

<h1>Привет, я могу содержать весь твой HTML-код!</h1>

Хорошая новость! Если вы используете Flask, вам не нужно подключать Jinja.Просто добавьте render_template при импорте Flask.Скопируйте и выполните следующий код!

from flask import Flask, render_template # подключаем render_template, который включает функции для Jinja

app = Flask(__name__)

@app.route("/")
def template():
    output = render_template("test.html") # рендерим шаблон return output # возвращаем то, что отрендерилось

app.run()

Теперь вы знаете, как избавиться от всего этого HTML кода в главном файле приложения. Круто, правда?

Дальше мы научимся работать с переменными.

Использование переменных в файле шаблона

В прошлом шаге мы создали в папке templates файл шаблона test.html простой шаблон.

Отредактируем его, заменив текст на:

<h1>Hello {{name}}, welcome to the {{place}}</h1>

Теперь вернемся к главному файлу нашего приложения и отредактируем его, передав переменные в шаблон!

from flask import Flask, render_template

app = Flask(__name__)

@app.route("/")
def template():
    output = render_template('test.html',name="Alex",place="Lab") # рендерим шаблон, передавая переменныеreturn output

app.run()

Несколько шаблонов

Для простоты примера не будем использовать переменные. Так бывает – у контентных страниц, например, /about, логики может вообще не быть! Сначала создайте в папке два файла с шаблонами:

 ### index.html
<h1>Здесь будет главная страница</h1>
<a href="/about"> О проекте </a>

### about.html
<h1>Здесь будет страница "О сервисе"</h1>
<a href="/"> На главную </a>

Затем измените главный файл app.py так:

from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def main():
    return render_template('index.html')


@app.route('/about')
def about():
    return render_template('about.html')


app.run()

Должна получиться примерно такая структура:

Project_folder

– templates

–– index.html

–– about.html

– app.py

Запустите приложение.

Вы увидите две страницы, соединенные ссылками, отлично!

Переменные, списки и словари

Поскольку мы не всегда работаем только со строковыми, численными или логическими переменными, но также со словарями или списками, для них также предусмотрены специальные элементы разметки.

Переменная

Чтобы использовать переменную в шаблонах, добавьте конструкцию {{ имя_переменной }} в html коде.

Предположим, вы передаете в шаблонизатор переменные title и description.

Тогда просто обратитесь к переменной по имени. Вам не нужно использовать print. Фигурные скобки и есть print.

Список

Чтобы использовать список в шаблонах, добавьте конструкцию {{ Список[индекс] }} в html коде.

Предположим, вы передаете в шаблонизатор список students.

Конечно, в реальных проектах так не делают, предпочитая проходить списки циклом, мы рассмотрим позже как это делать, но пример все равно работает:

<ul>
  <li>{{ students[0] }}</li>
  <li>{{ students[1] }}</li>
  <li>{{ students[2] }}</li>
</ul>

Словарь

Чтобы использовать словарь в шаблонах, добавьте конструкцию {{ имя_словаря.ключ }} или {{ имя_словаря['ключ'] }}  в html коде. В шаблоне две эти записи делают одно и то же, берут значение по ключу "ключ" из словаря. Запись через точку это синтаксический сахар шаблонизатора, его можно использовать только в шаблоне.

Предположим, вы передаете в шаблонизатор словарь page с ключами title и description.

Тогда используйте выражение с точкой:

<h2>{{ page.title }}</h2>
<p>{{ page.description }}</p>

Или можно записать то же самое без сахара.

<h2>{{ page['title'] }}</h2>
<p>{{ page['description'] }}</p>

Какой синтаксис использовать — вопрос вкуса и предпочтения программиста.

Области видимости

Обратите внимание, переменную нужно явно объявить и передать в шаблон. Переменные, не переданные в шаблонизатор явно, недоступны для вывода внутри шаблона.

Например, в этом примере мы передаем в шаблонизатор переменную name со значением Steven:

output = render_template("hello.html", name="Steven")

В этом примере, мы передаем в шаблонизатор список с элементами 122, 344, 720:

output = render_template('test.html', mylist=[122, 344, 720])

А в этом примере – словарь:

output = render_template('test.html', cat={"angry":False, "color":"grey", "name":"Sofia"})

Резюме

Шаблонизаторы позволяют удобно вынести HTML из основного файла.Шаблон – это HTML-файл с добавлением специальной разметки.Jinja – шаблонизатор, встроенный во Flask.Специальная разметка может содержать:– Переменные.– Ветвление.– Циклы.– Вызов функций.

Для вызова Jinja для обработки файла:

– подключите from flask import Flask, render_template;

– используйте return render_template(шаблон, переменные).

Чтобы использовать переменную, используйте конструкцию:

{{ variable1 }} {{ variable2 }}

Чтобы использовать список, используйте конструкцию:

{{ list[0] }} {{ list[1] }}

Чтобы использовать словарь, используйте конструкцию:

{{ dict.key1 }} {{ dict.key2 }}
