1.21 Flask - дебаггинг

Flask debug

В состав Flask входит простой сервер приложения, который нужен для локальной разработки и отладки. Если мы передадим в параметр запуска аргумент app.run(debug=True) , то наш локальный сервер будет автоматически перезапускать код при его изменении (авторелоад) и показывать его трейсбек в случае необработанного исключения прямо на страничке.

Важно: никогда не оставляйте дебаг включённым на продакшене, так как это серьёзная уязвимость.

Flask-DebugToolbar

Для вывода полезной информации прямо в шаблон можно воспользоваться дополнением Flask-DebugToolbar.

https://flask-debugtoolbar.readthedocs.io/en/latest/

Устанавливать его так же как и остальные дополнения для Flask.

Bug

В программировании есть несколько типов ошибок. Когда ошибка приводит к исключению и когда ошибка логическая.

В случае не отработанного программистом исключения в Python исполнение программы останавливается, и в stderr выводится подробное сообщение об ошибке. Называется это traceback (трассировка или трэйс), и в нём обычно содержится информация которая поможет нам понять в чём дело и что нужно исправить.

Как читать traceback

Например, у нас есть простой роут, который вычисляет что-то на сервере и возвращает клиенту.

@app.route("/divide/<int:x>/<int:y>/")
def divider(x, y):
    result = x / y
    return str(result)

Но если перейти по ссылке http://127.0.0.1:5000/divide/42/0/ то возникнет исключение, так как на ноль делить нельзя.

Вот последние строки нашего исключения, в них обычно находится строка которая и вызвала исключение.

File "/Users/vlad/PycharmProjects/stepik-qa/app.py", line 23, in divider
    result = x / y
ZeroDivisionError: division by zero

Полный текст исключения содержит всю цепочку вызовов (call stack), который наша программа сделала в момент исключения. Самый последний вызов внизу (most recent call last). Для каждого из вызовов написан путь к файлу где вызов сделан и номер строки в этом файле.

В нашем простом случае найти ошибку и исправить её не составляет труда.

Но в реальном мире ошибки прячутся лучше.

Основы дебага

В Pycharm есть встроенный дебаггер. Дебаггер - это программа, которая позволяет останавливать код в нужных нам местах, чтобы мы могли смотреть как именно он исполняется. Дебаггер не убирает баги, а только показывает их в замедленной скорости.

Для примера модифицируем наш роут, чтобы с первого взгляда не было понятно где затаился баг.

@app.route("/divide/<int:x>/<int:y>/")
def divider(x, y):
    result = 0for i in range(x, y + 1):
        result += 1 / (x - i + 42)
    return str(result)

Не особо вникайте в логику расчёта результата, она искусственная. Пример просто показывает что ошибка возникает не всегда, и с первого взгляда не понятно где она. Например, при таких параметрах http://127.0.0.1:5000/divide/0/22/ будет результат, а при http://127.0.0.1:5000/divide/0/422/ - исключение.

Полный код примера, на котором далее мы будем учиться запускать и пользоваться дебаггером.

from flask import Flask

app = Flask(__name__)


@app.route("/divide/<int:x>/<int:y>/")def divider(x, y):
    result = 0for i in range(x, y + 1):
        result += 1 / (x - i + 42)
    return str(result)


app.run(debug=True)

Каждый раз когда программа, запущенная через дебаггер, попадет в нашу строку кода, она остановится и дебаггер покажет нам информацию о текущем состоянии.

Основные возможности дебаггера

Дебаггер облегчает отладку.

Например, мы можем смотреть и менять значения любой переменной.

Все в том же примере нажмём правую кнопку мышки на любой переменой из раздела Variables:

Мы можем заменить значение переменной не перезапуская нашу программу! Или скопировать его. Попробуйте поменять x на 42 и посмотреть что получится.

Так же можно посмотреть все брейкпоинты, которые мы поставили в проекте.

При этом доступна подробная настройка каждого брейкпоинта. Вот эта кнопка:

Если по уже установленному брейкпоинту нажать ПКМ, откроется небольшое меню с его настройками:

Самое важное тут - это поле Condition (Conditional breakpoint). В нём мы можем записать условие, и брейкпоинт будет срабатывать только если это условие истинно. Очень удобно для поиска проблем связанных с логикой, без перезапуска программы для добавления отладочных if'ов. При задаче условия возле иконки брейкпоинта будет маленький знак вопроса, обозначающий что он задан с условием.

Теперь мы можем найти момент когда присходит деление на ноль.

Evaluate Expression

Более того, дебаггер позволяет нам в момент остановки на брейкпоинте даже исполнить любой код на питоне в контексте места где мы остановились! За это отвечает вот эта кнопка:

В поле мы можем написать строку питон-кода, и по нажатию кнопки Evaluate она исполнится, прямо как будто мы её написали сразу в коде функции.
