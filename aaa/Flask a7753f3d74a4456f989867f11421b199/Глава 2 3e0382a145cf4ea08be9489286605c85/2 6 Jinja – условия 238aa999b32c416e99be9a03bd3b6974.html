2.6 Jinja – условия

Шаблонизатор позволяет хранить шаблоны отдельно. Когда он получает данные, то подставляет их в нужные места шаблона. Так мы отдельно вычисляем данные и отдельно выводим их с помощью шаблона,  НЕ смешивая логику и представление. Когда мы хотим отредактировать логику, мы не трогаем шаблон, когда хотим отредактировать представление – работаем только с файлом шаблона. Это особенно удобно, когда речь идет про вывод больших блоков кода!

Взгляните на типичный шаблон интернет магазина. Вот какие блоки, использующие условия  и циклы могут быть на нем!

У шаблонизатора Jinja присутствуют мощные инструменты для условий и циклов. Мы рассмотрим лишь некоторые, дополнительные есть в официальной документации: https://jinja.palletsprojects.com/en/2.10.x/templates/ .

В этом топике вы будете решать задачи, связанные с приложением по доставке.Если вы проголодались – пропустите этот топик и отправляйтесь покушать :)

В этом топике вы научитесь:

– Выводить разные блоки в зависимости от данных, например, "скидка", "распродажа", "новинка", "топ".– Показывать / прятать блоки информации в зависимости от данных, например, профиль или панель администратора.

Работа с условиями

Мы применяем условие IF в шаблонизаторе, когда нам нужно показать / скрыть блок в зависимости от данных:– показать панель администратора только администраторам;– показать "нет товаров в корзине", если их нет;– показать "нет доступа", если доступа нет;– дать возможность удалять комментарий только его автору.

# предположим, в словаре user для обозначения является ли он администратором есть ключ is_admin, # значение которого может быть True или False

{% if user.is_admin %}
   <a href="#edit">Edit post</a>
{% endif %}

Мы применяем условие IF ... ELSE в шаблонизаторе, когда нам нужно показать разные блоки для двух случаев:– показать по-разному обычных и про-пользователей;– показать специальную цену, если клиент постоянный и обычную, если обычный;– показать профиль авторизованным и кнопку входа неавторизованным.

# предположим, в словаре user для обозначения того, залогинен ли пользователь, # есть ключ is_logged, значение которого может быть True или False

{% if user.is_logged %}
   <a href="#profile">{{ user.name }}</a>
{% else %}
   <a href="#auth">Login</a>
{% endif %}

Мы применяем условие IF ... ELIF  в шаблонизаторе, когда нам нужно показать разные блоки для каждого случая:– показать теги "новое","скидка","топ";– показывать разные блоки разным ролям пользователей (админам, модераторам, пользователям, гостям);– выводить разные состояния – "вкл, недоступен, удален".

# предположим, в словаре user есть ключ state для обозначения доступности пользователя, # значение которого может быть online или away или offline или какое то еще.

{% if user.state=="online" %}
   Онлайн
{% elif user.state=="away" %}
   Отошел
{% elif user.state=="offline" %}
   Недоступен
{% endif %}

Теперь попробуйте самостоятельно решить несколько задач на использование условий.

Резюме

Чтобы создать условие, используйте:

{% if условие %}
   блок кода
{% endif %}

Если у вас больше 2 вариантов

{% if условие %}
   блок кода
{% else %}
   блок кода
{% endif %}
{% if условие %}
   блок кода
{% elif %}
   блок кода
{% else %}
   блок кода
{% endif %}
