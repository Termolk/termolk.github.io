2.11 Jinja: Создание ссылок в шаблонах

На вашем сайте есть роуты, каждый со своим адресом.

И еще на вашем сайте есть ссылки из меню, списка элементов или поиска, которые на эти роуты ведут. В этом топике мы потренируемся создавать ссылки, а если точнее, использовать циклы в Jinja для создания ссылок.

Помните, что создавая новое представление для роута, рекомендуется указывать "слеш" в начале и в конце адреса, а при создании ссылок также добавлять "слеш" в конце

Вы научитесь:

  • создавать меню для сайта;
  • отмечать текущий пункт меню активным;
  • выводить списки на разделы или теги;
  • выводить ссылки из поиска на страницы конкретных элементов.

Меню для сайта

Обычное меню – просто набор ссылок, упакованных в <nav>.

Иногда для разметки меню используют неупорядоченный список <ul>.

Пункты для меню могут храниться в списке. В этом случае, нам нужно открыть тег меню, запустить цикл по этому списку, вывести в нем текст и адрес ссылки, а затем закрыть меню.

{% for link in links %}
   <a href="{{ link.link }}">{{ link.title }}</a>
{% endfor %}

Текущий пункт меню

Частая задача - отметить, на какой странице находится пользователь – например выделить цветом, рамкой или жирностью.

В верстке для этого обычно используется class=active.

Осталось только написать логику, когда его выводить.

Во Flask переменная запроса доступна в шаблонах, используйте request.path, чтобы проверить ваши ссылки!

{% for link in links %}
   <li><a href="..." class="{% if request.path == link.link %}active{% endif %}">
			...
      </a></li>
{% endfor %}

Или вы можете использовать set, тогда код будет более читаемый:

{% for link in links %}
   <li>
      {% if request.path == link.link %}{% set c = "active" %}
			{% else %} {% set c="" %} {% endif %}
      <a href="..." class="{{ c }}">...</a></li>
{% endfor %}

Список разделов

Бывает так, что содержимое сайта (еда, сотрудники, туры) сгруппировано по категориям и вам нужно вывести список на страницу категории. Примеры таких ссылок:

  • для еды
<a href="/drinks/">Напитки</a>
<a href="/streetfood/">Стритфуд</a>
<a href="/lunches/">Обеды</a>
  • для сотрудников
<a href="/management/">Менеджмент</a>
<a href="/designers/">Дизайнеры</a>
<a href="/developers/">Разработчики</a>

При этом список разделов обычно хранится в таком виде:

categories = [
  {"id":"drinks","title":"Напитки"},
  {"id":"streetfood","title":"Стритфуд"},
  {"id":"lunches","title":"Обеды"}
]

Чтобы вывести работающие ссылки на разделы, добавляем слеши к id в адресе:

{% for cat in categories %}
   <li><a href="/{{ cat.id }}/">{{ cat.title }}</a></li>
{% endfor %}

Получаем:

<a href="/drinks/">Напитки</a>
<a href="/streetfood/">Стритфуд</a>
<a href="/lunches/">Обеды</a>

Ссылки на внутренние страницы

Еще один случай, который иногда встречается: элемент находится в категории, а ссылку мы хотим дать с главной страницы (например, товар у нас лежит в разделе или страна относится к континенту или тур относится к стране).

Мы должны сформировать ссылку типа:

<a href="/категория/товар/"> ... </a>

Как это делается?

Вот, например, у нас есть список товаров:

items = [
 {"id":"espresso-machine123","name":"Кофеварка Пш-Пш 3","cat":"coffee-machines"},
 {"id":"vacuum-cleaner99","name":"Пылесос торнадо 9","cat":"vacuum-cleaners"},
 {"id":"lavalamp-66","name":"Лавалампа в цветочек","cat":"lamps"},
]

Тогда сперва мы используем в создании ссылки идентификатор категории (cat), а затем идентификатор товара (id) и получаем ссылку на внутреннюю страницу.

<nav>
{% for item in items %}
    <a href="/{{ item.cat }}/{{ item.id }}/">...</a>
{% endfor %}
</nav>

В результате мы получим такой набор ссылок:

<a href="/coffee-machines/espresso-machine123/">...</a>
<a href="/vacuum-cleaners/vacuum-cleaner99/">...</a>
<a href="/lamps/lavalamp-66/">...</a>

То, что нужно. При необходимости так можно собирать ссылки любой длины!

Множественное предпочтительнее единственного

Почему названия сущностей (lamps, coffee-machines) идут во множественном числе?

Дело в том, что, когда поисковые системы анализируют сайт, они строят свое понимание того, как он устроен на основе адресов. Чтобы однозначно дать понять, что страничка lamp 66 входит в раздел lamps, адрес странички lamp 66 должен включать в себя адрес раздела lamps

Правильное написание:

/lamps/

/lamps/lamp-66

Менее правильное написание:

/lamp/  (страница с разделом ламп называется "лампа")

/lamp/lamp-66

Совсем неправильное написание:

/lamps/

/lamp/lamp-66 (страница лампы не является дочерней для раздела с лампами)

Типичные ошибки

1. Терять слеш в начале

Предположим мы находимся на странице /about и хотим разместить на ней ссылки. Полезно помнить, что относительные ссылки в html могут указывать путь от корня href="/way" и от текущей страницы href="way" Бывают ещё абсолютные ссылки вида http://this.is/way, но в наших проектах мы не будем их использовать.

При нажатии на ссылку от корня ( на сайте this.is/about нажимаем на ссылку "/way" мы перейдем на this.is/way, все верно. А вот при нажатии на "way" мы перейдем на this.is/about/way , что выдаст нам ошибку 404.

Вывод: не забывайте слеш, если пишете путь от корня.

{% for link in links %}
   <li><a href="..." class="{% if request.path == link.link %}active{% endif %}">
			...
      </a></li>
{% endfor %}
