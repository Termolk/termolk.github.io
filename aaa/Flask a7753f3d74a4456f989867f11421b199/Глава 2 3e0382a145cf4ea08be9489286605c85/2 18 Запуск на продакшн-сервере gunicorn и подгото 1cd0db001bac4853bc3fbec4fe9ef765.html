2.18 Запуск на продакшн-сервере gunicorn и подготовка к переезду

Запуск на продакшн-сервере gunicorn

Когда вы запускаете сервер Flask, он предупреждает вас, чтобы вы не использовали его в боевом режиме, помните?

.

Gunicorn – отличный продакшн-сервер, отличия от сервера самого Flask будут для вас незаметны.

Давайте установим его, запустим и попробуем использовать! При публикации в интернете именно gunicorn  будет нашим сервером!

Запускаем приложение с помощью Gunicorn

1. Перейдем в терминал и установим gunicorn:

$ pip install gunicorn  (или используйте pip3)

2. Отредактируем наш файл app.py

У нас есть строка, которая запускает Flask-сервер.

Заменим ее на:

if __name__ == '__main__':
    app.run()

3. Запустите ваше приложение

Внимание, сервер gunicorn НЕ работает на Windows.Если у вас эта операционная система, пожалуйста,  переходите к следующему степуСделанного будет достаточно, чтобы gunicorn запустился на удаленном сервере!

Запустите приложение app, набрав в терминале :

$ gunicorn app:app

Gunicorn сообщит, что сервер поднят и работает где-то в районе http://0.0.0.0:8000/ или http://localhost:8000/ .

4. Откройте сайт в браузере.

Обратите внимание, gunicorn использует порт 8000 вместо 5000.

5. Все работает? Значит мы еще на шаг приблизились к переезду в онлайн!

6. Обновите ваш requirements.txt, иначе деплой на хероку не сработает.

Подготовка Procfile

Создайте специальный файл, который нужен для разворачивания на сервере –  Procfile.

Файл должен лежать в корне проекта, не иметь расширения и начинаться с большой буквы.

Содержимое файла должно быть таким:

web: gunicorn app:app

При разворачивании эти инструкции подскажут, какой сервер использовать и, какое приложение на нем запустить.

Проверьте себя:

– Файл должен начинаться с большой буквы.

– Файл должен называться именно Procfile.

– Файл должен содержать именно строку, приведенную выше с сохранением всех пробелов.

– Файл нужно включить в репозиторий, сделать коммит и отправить на гитхаб!
