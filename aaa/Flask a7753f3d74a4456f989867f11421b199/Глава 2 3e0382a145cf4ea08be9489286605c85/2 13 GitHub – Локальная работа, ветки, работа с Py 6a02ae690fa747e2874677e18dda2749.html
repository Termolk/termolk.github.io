2.13 GitHub – Локальная работа, ветки, работа с PyCharm

Локальная работа

Модель работы

Представьте, у вас есть репозиторий, размещенный на GitHub с проектом, который вы собираетесь развивать. Это может быть разветвленный репозиторий или оригинальный, суть в том, что у вас есть полный доступ к нему.

Стандартный подход к работе с проектом состоит в том, чтобы иметь локальную копию репозитория и фиксировать ваши изменения в этой копии, а не в удаленном репозитории, размещенном на GitHub. Этот локальный репозиторий имеет полную историю версий проекта, которая может быть полезна при разработке без подключения к интернету. После того, как вы что-то изменили в локальном, вы можете отправить свои изменения в удаленный репозиторий, чтобы сделать их видимыми для других разработчиков.

На рисунке ниже показана модель работы с GitHub.

Модель работы с GitHub

На этом рисунке показаны две отдельные области: GitHub и ваш компьютер.

Сначала рассмотрим область GitHub. В нем есть два хранилища:

  • upstream — это оригинальный репозиторий проекта, который вы скопировали,
  • origin — ваш fork (копия) на GitHub, к которому у вас есть полный доступ.

Чтобы перенести изменения с вашей копии в исходный репозиторий проекта, вам нужно сделать запрос на извлечение.

Если вы хотите внести небольшие изменения в свою копию (fork), вы можете использовать веб-интерфейс GitHub. Однако такой подход не удобен при разработке программ, поскольку вам часто приходится запускать и отлаживать их локально. Стандартный способ — создать локальный клон удаленного репозитория и работать с ним локально, периодически внося изменения в удаленный репозиторий.

Установка Git

Чтобы продолжить изучение этого руководства, вам нужно установить Git локально, если вы еще этого не сделали. Просто следуйте инструкции для вашей операционной системы (перевод – вот). Если вам почему-то не нравится это руководство, вы можете просто зайти в Google.

Чтобы убедиться, что Git был успешно установлен, введите команду ниже в терминале, чтобы отобразить текущую версию вашего Git:

git version

Если она сработала, давайте добавим в настройки Git ваши имя, фамилию и адрес электронной почты, связанный с вашей учетной записью GitHub:

git config --global user.name <YOUR_NAME>
git config --global user.email <EMAIL>

Создаем репозиторий на GitHub

В правом верхнем углу, рядом с аватаром есть кнопка c плюсиком, нажимая которую мы переходим к созданию нового репозитория.

Имя репозитория.

Оно может быть любое, необязательно уникальное во всем GitHub, потому что привязано к вашему аккаунту, но уникальное в рамках тех репозиториев, которые вы создавали.

Описание (Description)

Можно оставить пустым.

Public/private

выбираем открытый (Public), НЕ ставим галочку “Initialize this repository with a README” ( В README потом будет лежать какая-то основная информация, что же такое ваш проект и как с ним работать)..gitignore и LICENSE можно сейчас не выбирать.

Нажимаем кнопку Create repository.

Отлично, ваш репозиторий готов!

Использование VCS в PyCharm

Связываем репозиторий на GitHub c локальным репозиторием

Хотя для работы с GitHub профессиональные разработчики чаще используют терминал, мы начнем наши первые шаги с использования графического интерфейса, предоставленного PyCharm. У нас уже есть отдельно код, который мы написали для нашего проекта и созданный удаленный репозиторий на GitHub. Пора подружить их вместе. )

1. Получите ссылку на репозитоий.

Откройте страницу репозитория на Github, скопируйте ссылку:

2. Выберите систему контроля версий для интеграции.

Откройте  VCS > Enable version control integration в PyCharm.

Выберите Git в появившемся окошке.

3. Добавьте удаленный репозиторий.

Откройте VCS > Git > Remotes.

Нажмите +, чтобы добавить удаленный репозиторий, скопируйте в поле ввода ссылку на него.

4. Сделайте коммит файлов

Откройте VCS > Commit.

Поставьте галочки напротив всех файлов проекта .Не отмечайте папку .idea и другие вспомогательные файлы.Напишите сообщение к коммиту. Отлично подойдет "Initial commit".Нажмите Commit.

5. Отправьте коммит в удаленный репозиторий

Откройте Git > Push.

Подтвердите отправку!

6. Посмотрите на результат!

Ваш код опубликован на GitHub.  Вы почти настоящий разработчик! Теперь вы можете:

– Показать его друзьям-программистам.– Показать его преподавателю.– Использовать репозиторий для автоматического деплоя.Что делать, если вы забыли добавить какой-то файл?Ничего страшного, сделайте еще один коммит!

Резюме

Git нужен, чтобы:

– записывать состояние проекта;

– ходить туда-сюда по сохранениям;

– хранить свой код онлайн;

– делиться кодом с друзьями;

– еще много всего. )

GitHub – отдельный сервис, чтобы пользоваться git и социальными возможностями.Чтобы пользоваться git, его нужно установить. Проверьте, установлен ли он с помощью git --version .

Чтобы создать новый репозиторий для существующего проекта– перейдите на github.com;– нажмите «+» в правом верхнем углу;– затем задайте имя репозитория, нажмите «Создать»;– скопируйте ссылку на репозиторий .

Чтобы использовать контроль версий в PyCharm

– получите ссылку, нажав Clone or Download на вкладке Code;

– в Pycharm выберите пункт VCS;

– перейдите в VCS/Git/remotes;– нажмите + и скопируйте ссылку в появившееся окно. Нажмите Ок

;– перейдите в VCS/Commit;

– поставьте галочки напротив файлов и папок, которые хотите добавить в коммит;

– нажмите Commit;

– перейдите в VCS/Push;

– подтвердите отправку коммита в удаленный репозиторий;

– после получения подтверждающего сообщения, проверьте на GitHub, появились ли файлы. )
