2.9 Jinja – базовые шаблоны

На страницах сайта повторяется один и тот же код шапки сайта и подвала сайта.  Повторять его в  в шаблонах для каждой страницы - плохая идея. Это не очень удобно: если вам понадобится изменить шапку, то придется редактировать несколько файлов. Это не очень правильно: нарушается принцип DRY (Don't Repeat Yourself) https://ru.wikipedia.org/wiki/Don%E2%80%99t_repeat_yourself .

Шаблонизатор Jinja позволяет создать один общий базовый шаблон, а уже в него вкладывать шаблоны страниц, таких как главная, каталог, контакты и так далее.

В этом уроке вы научитесь:

– выносить повторяющийся код в отдельный шаблон.

У шаблонизатора есть и более мощные возможности.

Больше информации здесь: https://jinja.palletsprojects.com/en/2.10.x/templates/#template-inheritance .

Принцип действия

Шаблонизатор Jinja позволяет создать один общий базовый шаблон (для этого вам нужно будет взять одну из страниц и вырезать из нее содержательную часть), а уже в него "вклеивать" шаблоны страниц, таких как главная, список, поиск и так далее.

Таким образом, ваш список шаблонов для сайта с вакансиями будет выглядеть как-то так:

– общий шаблон (шапка, подвал, стили, скрипты)

– специфичная часть шаблона страницы одной вакансии

– специфичная часть шаблона страницы списка вакансий

– специфичная часть шаблона страницы поиска

– специфичная часть шаблона главной

– и так далее

На самом деле этот механизм реализован через расширение шаблонов и может выглядеть необычно: в Jinja шаблон страницы получает контроль и "вклеивает себя" в общий шаблон, обычно обзаводясь шапкой, и подвалом.

Присматриваемся к страницам

Посмотрите на пример кода ниже. Это упрощенный код страницы сайта со списком вакансий. Конечно, страница настоящего сайта сложнее, а ее код длиннее, но структура остается примерно такой же.

<!DOCTYPE html>
<html>
	<head>
		<title>Джуманджи вакансии</title>
		<script src="..."/><style link=".."/>
	</head>
	<body>
		<header>
			<nav>menu</nav>
		</header>
		<div class="container">
			<div class="content">
				<h2>Вакансии</h2>
				<div class="vacancies">
	        (list of vacancies)
			  </div>
			</div>
		</div>
		<footer>site footer</footer>
	</body>
</html>

А следующий пример – упрощенный код страницы одной вакансии.

<!DOCTYPE html>
<html>
	<head>
		<title>Джуманджи вакансии</title>
		<script src="..."/><style link=".."/>
	</head>
	<body>
		<header>
			<nav>menu</nav>
		</header>
		<div class="container">
			<div class="content">
				<h2>Одна вакансия</h2>
				<div class="vacancy-details">
	        (list of vacancies)
			  </div>
			</div>
		</div>
		<footer>site footer</footer>
	</body>
</html>

Можно заметить, что специфичной для страницы является часть между тегами <div class="container"></div>, а остальное повторяется от страницы к странице. Давайте вынесем повторяющиеся блоки в отдельный файл!

Создаем базовый шаблон

1. Создайте файл base.html, оставьте там код, который повторяется от страницы к странице.

Код, который не повторяется, сохраните отдельно, он понадобится нам позже, а пока замените его на:

{% block container %} {% endblock %}

У вас получится примерно такой шаблон:

<!DOCTYPE html><html><head><title>Videos – Stepik Media</title><script src="..."/>
  <style link=".."/>
</head>
<body>

  <header><nav>menu</nav></header>

  <div class="container">
    {% block container %} {% endblock %}
  </div><footer>
    site footer
  </footer></body></html>

2. Создайте файл шаблона

Теперь возьмите код, который вырезали и продолжайте работать с ним:

– Добавьте в начале инструкцию, что ваш шаблон нужно будет "вклеить" в базовый шаблон base.html:

{% extends 'base.html' %}

– Оберните шаблон в конструкцию {% block container %}, чтобы показать, какую именно часть нужно будет "вклеить":

{% block container %}

  ...

{% endblock %}

Результат должен получиться примерно таким:

{% extends 'base.html' %}

{% block container %}

   <div class="content"><h2>Videos</h2><div class="videos">
        (list of videos)
     </div></div>

{% endblock %}

3. Проверьте себя

– имя файла базового шаблона xxx.html должно совпадать с инструкцией {% extends 'xxx.html' %} в дочернем шаблоне;– название блока в базовом шаблоне {% block yyy %} должно совпадать с названием блока дочернем шаблоне {% block yyy %}.

Include

Также повторяющийся на нескольких страницах HTML код можно вынести в отдельный файл и импортировать в шаблон с помощью include:

{% extends 'base.html' %}

{% block container %}

   {% include 'banner.html' %}

   <div class="content"><h2>Videos</h2><div class="videos">
        (list of videos)
     </div></div>

{% endblock %}

Super

Обычно в общем шаблоне блок, куда вклеивается шаблон представления, будет пустым, так как его содержимое все равно бы заменилось содержанием блока представления. Вот так:

# base.html

<html><body>
    {% block title %}
            Мой сайт с вакансиями.
    {% endblock %}
</body></html>
#  categories.html

{% extends "base.html" %}
{% block title %}
    Вот такие типы вакансий у нас есть:
{% endblock %}

В результате представление, использующее шаблон categories.html, отрендерится так:

<html><body>
    Вот такие категории у нас есть:
</body></html>

Как мы видим, контент, который был в базовом шаблоне внутри {% block title %}, исчез из получившегося html.

Однако, мы можем использовать конструкцию {{ block.super }}, чтобы сохранить его. Вот так:

#  categories.html

{% extends "base.html" %}
{% block title %}
    {{ block.super }} Вот такие категории у нас есть:
{% endblock %}

В результате получится:

<html><body>
    Мой сайт с вакансиями. Вот такие категории у нас есть:
</body></html>

Не уверены, что вам это пригодится в ближайшее время, но, как минимум, это весело. :)

Резюме

Использование базовых шаблонов позволяет не повторяться, создать структуру шаблонов, упростить их чтение и редактирование.

Чтобы задать в базовом шаблоне блок для вклеивания дочернего, напишите:

{% block container %} {% endblock %}

Подготовьте дочерний шаблон к использованию вместе с базовым, оберните его в

{% extends 'base.html' %}

{% block container %}

  ...

{% endblock %}

Чтобы принудительно вклеить один шаблон в другой полностью, используйте тег include

{% include 'banner.html' %}
