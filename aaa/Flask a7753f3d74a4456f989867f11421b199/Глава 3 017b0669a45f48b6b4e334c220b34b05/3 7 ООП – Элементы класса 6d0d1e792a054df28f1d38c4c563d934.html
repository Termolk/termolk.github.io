3.7 ООП – Элементы класса

Вы уже знаете, что такое классы, как их создавать и как они используются в Python. Теперь давайте узнаем чуть больше об экземплярах классов.

Экземпляр класса это то же самое, что и объект. Если бы, например, у нас был класс River, мы могли бы поместить в него такие экземпляры, как Volga, Seine, Neil и тд. Все они имеют одинаковую структуру и разделяют все атрибуты класса, определенные в классе River.

Однако изначально все экземпляры класса будут идентичны друг другу. В большинстве случаев это не то, что нам требуется. И чтобы настроить начальное состояние экземпляра, используется метод __init__.

В этом топике вы научитесь:

– использовать метод __init__

– пользоваться параметром self

– задавать и вызывать атрибуты экземпляра

def __init__()

Метод __init__ является конструктором. Конструкторы - это концепция объектно-ориентированного программирования. Класс может иметь один и только один конструктор. Если __init__ определен внутри класса, он автоматически вызывается при создании нового экземпляра класса. Возьмем наш класс River в качестве примера:

class River:# список всех рек
    all_rivers = []

    def __init__(self, name, length):
        self.name = name
        self.length = length
        # добавляем текущую реку в список всех рек
        River.all_rivers.append(self)

volga = River("Волга", 3530)
seine = River("Сена", 776)
nile = River("Нил", 6852)

# далее печатаем все названия рек
for river in River.all_rivers:
    print(river.name)
# Output:
# Волга
# Сена
# Нил

Мы создали три экземпляра (или объекта) класса River: volga, seine и nile. Поскольку мы определили параметры name и length для __init__, они должны быть явно переданы при создании новых экземпляров. Поэтому что-то вроде volga = River () может вызвать ошибку.

Метод __init__ указывает, какие атрибуты будут у  экземпляров нашего класса. В данном случае это name и length.

self

Возможно, вы заметили, что у нашего метода __init__ был другой аргумент, кроме name и length: self. Аргумент self представляет конкретный экземпляр класса и позволяет нам получить доступ к его атрибутам и методам. В примере с __init__ мы создаем атрибуты для конкретного экземпляра и присваиваем им значения аргументов метода. Важно использовать параметр self внутри метода, если мы хотим сохранить значения экземпляра для последующего использования.

В большинстве случаев нам также необходимо использовать параметр self в других методах, потому что при вызове метода первым аргументом, который ему передается, является сам объект. Давайте добавим метод к нашему классу River и посмотрим, как он будет работать. Синтаксис методов на данный момент не важен, просто обратите внимание на использование self:

class River:
    all_rivers = []

    def __init__(self, name, length):
        self.name = name
        self.length = length
        River.append(self)

    def get_info(self):
        print("Длина{0} равна {1} км".format(self.name, self.length))

Теперь, если мы применим этот метод к объектам, которые создали ранее, то получим:

volga.get_info()
# Длина Волги равна 3530 км
seine.get_info()
# Длина Сены равна 776 km
nile.get_info()
# Длина Нила равна 6852 km

Как видите, для каждого объекта метод get_info () печатает именно его значения, именно потому что мы использовали ключевое слово self в качестве первого аргумента в get_info ().

Обратите внимание, что когда мы не вызываем метод, мы не пишем аргумент self в скобках. Параметр self (который представляет конкретный экземпляр класса) передается методу экземпляра неявно при его вызове. Таким образом, существует два способа вызова метода экземпляра: self.method () или class.method (self). В нашем примере это будет выглядеть так:

# self.method()
volga.get_info()
# Длина Волги равна 3530 км
# class.method(self)
River.get_info(volga)
# Длина Волги равна 3530 км

Атрибуты экземпляра

Классы в Python имеют два типа атрибутов: атрибуты класса и атрибуты экземпляра. Вы уже должны знать, что такое атрибуты класса, поэтому здесь мы сосредоточимся на атрибутах экземпляра. Атрибуты экземпляра определяются в методах и хранят информацию, специфичную для экземпляра.

В классе River атрибуты name и length являются атрибутами экземпляра, поскольку они определены в методе __init__ и имеют перед собой self. Обычно атрибуты экземпляра создаются в методе __init__, поскольку он является конструктором, но вы можете определить атрибуты экземпляра и в других методах. Атрибуты экземпляра доступны только из области видимости объекта, поэтому такой код выдаст ошибку:

print(River.name)  # AttributeError

Атрибуты экземпляра, естественно, используются для различения объектов: их значения различны для разных экземпляров.

volga.name  # "Волга"
seine.name  # "Сена"
nile.name   # "Нил"

Поэтому, решая, какие атрибуты выбрать в вашей программе, вы должны сначала решить, хотите ли вы, чтобы она сохраняла значения, уникальные для каждого объекта, или, наоборот, значения, которые будут общими для всех экземпляров класса.

Резюме

В этой теме вы узнали об экземплярах классов.

Если классы - это абстракция, некоторый шаблон для похожих объектов,  то экземпляр класса - это своего рода пример конкретного объекта, который соответствует структуре, общей для этого класса. В вашей программе вы можете создавать столько объектов класса, сколько потребуется.

Для создания объектов с разными начальными состояниями классы имеют конструктор __init__ , который позволяет нам определять необходимые параметры для конкретного объекта. Ссылка на конкретный экземпляр в методах осуществляется через ключевое слово self . В методе __init__ мы определяем атрибуты экземпляра, которые будут различаться для всех экземпляров.

В этом топике вы научились:

Создавать экземпляры и задавать их атрибуты с помощью конструктора:

def __init__(self, name, length):
    self.name = name
    self.length = length
    River.all_rivers.append(self)

Вызывать метод экземпляра напрямую и через класс:

# self.method()
volga.get_info()
# Длина Волги равна 3530 км

# class.method(self)
River.get_info(volga)
# Длина Волги равна 3530 км

Как правильно обращаться к атрибуту экземпляра:

volga.name  # "Волга"
seine.name  # "Сена"
nile.name   # "Нил"

И как неправильно:

print(River.name) # AttributeError

По большей части в наших программах мы будем иметь дело не с классами напрямую, а с их экземплярами, поэтому очень важно знать, как их создавать и работать с ними!
