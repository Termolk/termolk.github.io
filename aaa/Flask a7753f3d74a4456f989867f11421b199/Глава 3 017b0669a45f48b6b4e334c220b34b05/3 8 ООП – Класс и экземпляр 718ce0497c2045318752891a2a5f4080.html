3.8 ООП – Класс и экземпляр

Вы уже знаете, чем в Python различаются атрибуты класса и атрибуты экземпляра. Как вы наверняка помните, атрибуты класса являются общими для всех объектов класса, а атрибуты экземпляра специфическими для каждого экземпляра. Более того, атрибуты класса определяются внутри класса, но вне каких-либо методов, а атрибуты экземпляра обычно определяются в методах, чаще всего в __init__ .

Теперь давайте рассмотрим разницу между атрибутами класса и атрибутами экземпляра более подробно.

Изменение атрибутов

Предположим, у нас есть класс Pet:

class Pet:
    kind = "mammal"
    n_pets = 0  # количество питомцев
    pet_names = []  # список имен всех питомцев

    def __init__(self, spec, name):
        self.spec = spec
        self.name = name
        self.legs = 4

Этот класс имеет три атрибута класса: kind, n_pets и pet_names, а также три атрибута экземпляра, определенных в методе __init__: spec, name и legs.

Давайте создадим несколько экземпляров этого класса и посмотрим, как работает изменение атрибутов класса или атрибутов экземпляра в Python:

tom = Pet("cat", "Tom")
avocado = Pet("dog", "Avocado")
ben = Pet("goldfish", "Benjamin")

Мы создали три экземпляра класса Pet, которые имеют одинаковые атрибуты класса и разные атрибуты экземпляра. Теперь имеет смысл изменить значение n_pets, потому что у нас более 0 питомцев. Так как n_pets является целым числом, которые относятся к неизменяемому типу данных, мы можем изменить его значение для всего класса, только если мы получим к нему доступ непосредственно как к атрибуту класса:

# получим доступ к атрибуту класса напрямую через класс
Pet.n_pets += 3

Pet.n_pets      # 3
tom.n_pets      # 3
avocado.n_pets  # 3
ben.n_pets      # 3

Если бы мы попытались изменить значение n_pets через экземпляры, то у нас бы ничего не получилось:

# попытаемся получить доступ к атрибуту класса через экземпляры
tom.n_pets += 1
avocado.n_pets += 1
ben.n_pets += 1

Pet.n_pets      # 0
tom.n_pets      # 1
avocado.n_pets  # 1
ben.n_pets      # 1

Несмотря на то, что все экземпляры имеют доступ к атрибуту класса, если эти атрибуты неизменны, изменение их значения для одного экземпляра не изменит их для всего класса.

То же самое будет и с атрибутом kind, поскольку строки также неизменны в Python. Если мы изменим его для объекта ben (поскольку золотая рыбка не является млекопитающим), он останется неизменным для других атрибутов:

ben.kind = "fish"

Pet.kind      # "mammal"
tom.kind      # "mammal"
avocado.kind  # "mammal"
ben.kind      # "fish"

В тех случаях, когда есть несколько уникальных объектов, которые должны иметь другое значение переменной класса, это вполне нормально. Однако, если таких объектов много, вам следует рассмотреть возможность сделать этот атрибут атрибутом экземпляра!

Ситуация с атрибутом pet_names другая. Атрибут pet_names является списком и, следовательно, изменяемым, поэтому изменения в нем влияют на весь класс:

tom.pet_names.append(tom.name)
avocado.pet_names.append(avocado.name)
ben.pet_names.append(ben.name)

Pet.pet_names      # ["Tom", "Avocado", "Benjamin"]
tom.pet_names      # ["Tom", "Avocado", "Benjamin"]
avocado.pet_names  # ["Tom", "Avocado", "Benjamin"]
ben.pet_names      # ["Tom", "Avocado", "Benjamin"]

Если по какой-то причине мы хотели, чтобы атрибут класса pet_names сохранял разные значения для разных экземпляров, мы могли бы создать новый список вместо добавления к существующему:

tom.pet_names = ["Tom"]
avocado.pet_names = ["Avocado"]
ben.pet_names = ["Benjamin"]

Pet.pet_names      # []
tom.pet_names      # ["Tom"]
avocado.pet_names  # ["Avocado"]
ben.pet_names      # ["Benjamin"]

Но это не кажется таким уж удобным или необходимым: в конце концов, это атрибут класса, и идея заключается в том, что он хранит значения, общие для всех экземпляров. Итак, еще раз, если вы хотите, чтобы какой-то атрибут хранил уникальные значения, сделайте его атрибутом экземпляра!

Как, например, переменная legs. Это атрибут экземпляра, даже если он явно не передается в качестве аргумента метода __init__. Значение по умолчанию равно 4, но мы можем изменить его, если потребуется. Это было бы полезно для объекта ben, потому что у рыб нет ног (у них есть плавники, но давайте оставим вопрос о том, может ли плавник считаться ногой на другой раз). Вот как мы меняем значение legs для объекта ben :

ben.legs = 0

Вот и все! В изменении атрибутов экземпляра нет ничего сложного, потому что, опять же, они влияют только на один объект.

Добавление атрибутов

В дополнение к изменению атрибутов мы также можем создавать атрибуты для класса или конкретного экземпляра. Например, мы хотим видеть информацию о всех видах наших питомцев. Мы могли бы записать ее в самом классе с самого начала или создать переменную следующим образом:

Pet.all_specs = [tom.spec, avocado.spec, ben.spec]

tom.all_specs      # ["cat", "dog", "goldfish"]
avocado.all_specs  # ["cat", "dog", "goldfish"]
ben.all_specs      # ["cat", "dog", "goldfish"]

Еще мы могли бы создать атрибут для конкретного экземпляра. Например, мы хотим вспомнить породу собаки под именем Avocado. Про породы чаще говорят применительно к собакам (у кошек тоже есть породы, но они не так сильно различаются), поэтому имеет смысл,  чтобы только у нее был атрибут с такой информацией:

avocado.breed = "corgi"

Здесь мы создали атрибут breed для объекта avocado и присвоили ему значение corgi. Другие экземпляры класса Pet , а также самого класса не имеют этого атрибута, поэтому следующие строки кода могут вызвать ошибку:

Pet.breed  # AttributeError
tom.breed  # AttributeErrorben.breed  # AttributeError

Заключение

В этой теме мы рассмотрели различия в использовании атрибутов класса и атрибутов экземпляра.

Атрибуты класса используются для хранения информации, свойственной для всех экземпляров класса, но их использование может оказаться сложным, если мы не примем во внимание тип переменной.

Итак, в каких случаях мы хотим использовать атрибуты класса? Ну, во-первых, если мы хотим определить значения по умолчанию для всех объектов. Во-вторых, хранить необходимые класс-специфические константы (например, математические). И, наконец, вести учет данных всех объектов, как в примере с pet_names. Возможно, вы захотите иметь легкий доступ к определенной информации каждого экземпляра вашего класса, и в этом случае вы можете использовать атрибут изменяемого типа.

Помните, что изменение значений атрибутов класса зависит от того, являются ли они изменяемыми или нет. Учитывайте это при написании вашей программы и работе с объектами класса!

Атрибуты экземпляра, с другой стороны, хранят информацию, различную для каждого экземпляра, и это, очевидно, их основная функция. Изменение и добавление новых атрибутов экземпляра может повлиять только на один объект, но, тем не менее, обращайте внимание на изменения, которые вы делаете.

Мы надеемся, что теперь вам понятна разница между атрибутами класса и экземпляра, и вы будете успешно использовать полученные знания в своих программах!
