3.4 Работа с файлами

До этого момента вы создавали программы, все данные для которых хранились в тексте самой программы (что не очень удобно, так как они занимают много места в коде) или получались от пользователя (что не очень удобно, так как они теряются, когда работа программы заканчивалась).

Часто бывает, что программе необходимо обрабатывать и хранить данные, расположенные снаружи: параметры конфигурации, некоторый набор данных для обработки, журналы выполнения и так далее. Самый простой способ хранения данных - использовать файлы, которые поддерживаются всеми современными операционными системами. Вы можете рассматривать файл как совокупность данных, которые хранятся на диске или другом устройстве и которыми можно манипулировать как единым целым, если обращаться к нему по имени. Файлы могут быть организованы в каталоги, которые действуют как папки для других файлов и каталогов.

Приемы работы с файлами, описанные в этой главе, позволят вам сохранять данные между вызовами программы, а также обмениваться данными между программами.

Вы научитесь:

  1. Читать строки из файлов.
  1. Записывать строки в файл.
  1. Работать с файлами в формате json.

Чтение из файла

Чтобы работать с файлами, нужно открыть файл и получить соответствующий объект файла.Если файл не может быть открыт, возникает ошибка OSError.

file - это подобный пути объект, задающий имя пути (абсолютное или относительное к текущему рабочему каталогу) открываемого файла или целочисленный дескриптор файла оборачиваемого файла. (Если указан файловый дескриптор, он закрывается, когда возвращаемый объект ввода-вывода закрывается, если для closefd не установлено значение False.)

mode - это необязательная строка, которая указывает режим, в котором открывается файл. По умолчанию используется значение 'r', что означает «открыт для чтения в текстовом режиме».

# Этот код выведет содержимое файла. Не забудьте сначала создать файл!

f = open('foo.txt', 'r')
contents =f.read()
f.close()

print(contents)

Функция open() открывает и возвращает дескриптор файла, который можно использовать для чтения или записи файла. Первый аргумент – имя файла, второй – режим. Вот, какие режимы бывают:

  • 'x' для эксклюзивного создания (будет исключение FileExistsError, если файл уже существует),
  • 'r' для чтения,
  • 'w' для записи,
  • 'a' для дополнения файла новыми данными, без удаления старых,
  • '+' для чтения/записи.

Метод f.close() закрывает файл. Рекомендуется закрывать файл, чтобы освободить к нему доступ, записать несохраненные изменения, освободить память.

Перезапись файла

Если мы запишем данные в файл, можно при следующем запуске прочитать их и использовать снова. Обратите внимание, при использовании метода write старое содержимое файла будет перезаписано.

# Этот код запишет строку в файл. Не забудьте сначала создать файл!

contents = "Я умею работать с файлами!"

f = open('foo.txt', 'w')
f.write(contents)
f.close()

Более современная форма записи

Если вам не нравится каждый раз закрывать файл – а кому это понравится? – используйте конструкцию with, которая называется менеджер контекста и файл закроется сам.

# чтение
with open("foo.txt", "r") as f:
   contents = f.read()

# запись
contents = "Я умею работать с файлами!"
with open("foo.txt", "w") as f:
   f.write(contents)

Работа с JSON файлами

В Python есть стандартная библиотека json, нам интересны следующие ее методы:

json.load()  # прочитать json из файла и конвертировать в python объект
json.loads() # тоже самое, но из строки с json  (s на конце от string/строка)

json.dump()  # конвертировать python объект в json и записать в файл
json.dumps() # тоже самое, но в строчку

Запись списка или словаря в файл:

import json

my_list = ['foo', 'bar']


# вариант 1
contents = json.dumps(my_list)
with open("foo.json", "w", encoding='utf-8') as f:
   f.write(contents)

# вариант 2with open("foo.json", "w", encoding='utf-8') as f:
   json.dump(my_list, f)

Чтение списка или словаря из файла:

import json

#вариант 1with open("foo.json", "r", encoding='utf-8') as f:
   contents = f.read()
my_list = json.loads(contents)

#вариант 2with open("foo.json", "r", encoding='utf-8') as f:
   my_list = json.load(f)

Аргумент encoding='utf-8' в open нужен для корректной работы с Unicode кодировкой utf-8 в Windows.

Библиотека pprint

В python есть стандартная библиотека pprint (от pretty print), которая умеет превращать python объект в "красивую" строчку. Это может быть полезно если у вас есть json объект записанный в файл и вы хотите разобраться как он устроен.

import pprint
import json
f = open('test.json', 'r')
data = json.load(f)f.close()

print(data)

Вернет

{'courses': {'1': {'name': 'python'}, '2': {'name': 'java'}, '3': {'name': 'css'}}, 'description': 'Some long text ....', 'subtitle': 'best site ever', 'title': 'Stepik Demo'}

Тогда как

pprint.pprint(data)

Вернет

{'courses': {'1': {'name': 'python'}, '2': {'name': 'java'}, '3': {'name': 'css'}}, 'description': 'Some long text ....', 'subtitle': 'best site ever', 'title': 'Stepik Demo'}
{'courses': {'1': {'name': 'python'},
             '2': {'name': 'java'},
             '3': {'name': 'css'}},
 'description': 'Some long text ....',
 'subtitle': 'best site ever',
 'title': 'Stepik Demo'}

Резюме

Работа с файлами позволит вам

  • хранить данные отдельно от логики,
  • сохранять данные между вызовами программы.

Чтобы получить строку из файла используйте

with open("foo.txt", "r") as f:
   contents = f.read()

Чтобы записать строку в файл, используйте

contents = "Я умею работать с файлами!"with open("foo.txt", "w") as f:
   f.write(contents)
