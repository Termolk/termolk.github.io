5.5 Flask: Регистрация

В первом топике про авторизацию мы научились узнавать пользователя и предоставлять или запрещать ему доступ в разные разделы сайта. Во второй части мы закопаемся в тему логинов и паролей немного глубже,  научимся регистрировать пользователей, использовать WTF и запрет доступа с помощью декораторов.

Вы научитесь:

  • Регистрировать нового пользователя;
  • Проверять данные формы.

Регистрация

Теперь сделаем упрощенную регистрацию.Нам понадобится представление, форма и страница, информирующая об успешной регистрации.Представление будет обрабатывать запросы на отображение формы и добавлять пользователя в базу данных.

Файл registration.html:

<h1>Регистрация</h1>

{% if error_msg %}
<div>
  {{ error_msg }}
</div>
{% endif %}

<form action="/registration" method="post">
  Имя: <input type="text" name="username" value="">
  Пароль: <input type="password" name="password"><button type="submit">Зарегистрироваться</button></form><a href="/login">Войти</a>

Подтверждение ввода пароля дважды удостоверяет пользователя, что не было опечатки. Другое решение - поле с возможностью режима просмотра введенного пароля, но это уже в области задач фронтэнда.

Так выглядит функция обработки запросов отвечающих за регистрацию:

Создадим страницу поздравления с регистрацией:

<h1>Регистрация</h1>

<p>Поздравляю, вы зарегистрированы под именем {{ username }} !</p>

Тестируем наше приложение. Для этого переходим со страницы логина на страницу регистрации. Вводим имя и пароль. В случае успеха мы оказываемся на странице поздравления и видим сообщение, что пользователь добавлен.

Получилось!

Однако, если мы более пытливы, то заметим, что именем пользователя и паролем может стать последовательность пробельных символов, что пользователь может ввести неправильный пароль и что кроме регистрации нам нужен сброс и смена пароля. Также, не обозначена минимальная и максимальная длина имени и пароля. Займемся улучшениями дальше, а пока - практика.

Проверка существования до регистрации

Некоторые пользователи не помнят, регистрировались ли они уже или нет.А некоторые по ошибке повторно отправляют форму. Чтобы устранить дубли, можно перед созданием пользователя проверять его существование.

Например:

#  Проверяем, есть ли такой юзернейм в базе
user = User.query.filter_by(username=username).first()

# Если такой пользователь существует if user:

    # Не можем зарегистрировать, так как пользователь уже существует
    error_msg = "Пользователь с указанным именем уже существует"return render_template("page_registration.html", error_msg=error_msg)

Совместное использование с WTF – регистрация

Изменим функцию регистрации с использованием WTF.

Создадим класс формы.

# Форма регистрации
class RegistrationForm(FlaskForm):# Добавляем поле имени пользователя
    username = StringField("Имя")

    # Добавляем поле пароля
    password = PasswordField("Пароль")

И используем форму в шаблоне!
