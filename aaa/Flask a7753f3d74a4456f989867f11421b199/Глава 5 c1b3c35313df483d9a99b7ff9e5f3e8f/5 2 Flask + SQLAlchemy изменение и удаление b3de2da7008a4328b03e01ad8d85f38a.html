5.2 Flask + SQLAlchemy: изменение и удаление

Изменение и удаление – важная часть работы.

В этом топике вы научитесь:

– Редактировать записи в базе– Удалять записи из базы

Будьте внимательны! Записи, удаленные из базы не возвращаются!

Изменение записи

Хорошая новость – вы уже умеете редактировать записи с помощью SQLAlchemy!

Изменение – это последовательность из:1) получение;2) изменение значения одного или нескольких полей;3) сохранение (коммит).Если у вас есть, например, студенты, которые заданы так:

class User(db.Model):
    __tablename__ = 'users'
    uid = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String(50))
    mail = db.Column(db.String(50))

И вы получаете одного из них с помощью запроса:

user = db.session.query(User).get(1)

Вы можете изменить его так:

user.mail = "mynewmail@gmail.com"

После этого сохранить изменения.

db.session.commit()

Массовое изменение

Еще вы можете использовать метод update, чтобы изменить сразу все выбранные записи.

Допустим у вас есть студенты и вы хотите добавить их в группу:

class Student(db.Model):
    __tablename__ = 'students'
    uid = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String())
    email = db.Column(db.String())
    group = db.Column(db.String())

Вы можете изменить его так:

student = db.session.query(Student).update({"group":"27"})

Удаление

Удаление – новая для вас операция, но она тоже оказывается простой.

Вы как будто говорите: база, пожалуйста, удали объект.

user = db.session.query(User).get(1)
db.session.delete(user)

Массовое удаление

Если хочется поменять сразу несколько записей по какому-то условию, это тоже можно сделать!

db.session.query(User).filter(User.age >= 25).delete(synchronize_session=False)

Status: deleted

Не всегда удаление пользователем приводит к удалению из базы.

Пользователи склонны удалять контент случайно или импульсивно, к тому же, удаление может разорвать связи, которые вам хотелось бы сохранить (например, вам нравится видеть список покупок пользователя, даже если он и не хочет его видеть).

Часто используется паттерн архивации вместо удаления.

Например, вместо удаления мы будем менять статус записи о покупке:

@app.route("/purchases/delete/<int:pur_id>/")
def delete_purchase(pur_id):
    purchase = db.session.query(Purchase).get_or_404(pur_id)
    purchase.status = "deleted"
    db.session.commit()

Тогда мы можем показывать пользователю только не удаленные покупки:

@app.route("/purchases/")
def render_purchases():
    purchases = db.session.query(Purchase).filter(Purchase.status!="deleted")

А сами – ура! – видеть его полную историю в режиме администратора как-нибудь так:

@app.route("admin/user/<int:uid>/purchases/")
def render_user_purchases_adminmode(uid):
    purchases = db.session.query(Purchase).filter(Purchase.user_id==uid)

Но будьте осторожны и не забудьте согласовать такие манипуляции с вашим юридическим отделом – тайное хранение данных пользователей не всегда может сойти вашей компании с рук, особенно, если ваш юридический отдел обещает всем "полное удаление данных".  :)

Резюме

Изменение:

student = db.session.query(Student).get(1)
student.mail = "mynewmail@gmail.com"
db.session.commit()

Удаление:

user = session.query(User).get(1)
db.session.delete(user)

или

user = session.query(User).get(1)
user.delete()

Вместо удаления мы может менять статус записи:

@app.route("/purchases/delete/<int:pur_id>/")
def delete_purchase(pur_id):
    purchase = db.session.query(Purchase).get_or_404(pur_id)
    purchase.status = "deleted"
    db.session.commit()
