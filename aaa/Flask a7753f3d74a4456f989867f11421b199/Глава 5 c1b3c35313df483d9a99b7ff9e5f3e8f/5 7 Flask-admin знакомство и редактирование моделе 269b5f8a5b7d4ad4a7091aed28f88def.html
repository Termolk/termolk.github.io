5.7 Flask-admin: знакомство и редактирование моделей

Flask-Admin решает скучную проблему создания интерфейса администратора поверх существующей модели данных. Без особых усилий он позволяет управлять данными вашего веб-сервиса через удобный интерфейс.

Клиентская часть работает поверх уже знакомого вам Twitter Bootstrap,  к сожалению, всего лишь версии 3.

В этом топике вы научитесь

  • Устанавливать и настраивать Flask-admin;
  • Редактировать контент через админку;
  • Настраивать отображение и права при работе с моделями.

Начинаем работу

Предположим, у нас есть приложение, в котором описаны модели и созданы таблицы.

Для того, чтобы построить админку к приложению, нужно:

1. Установить пакет flask-admin:

pip install flask-admin

2. Подключить Admin и ModelView:

from flask_admin import Admin
from flask_admin.contrib.sqla import ModelView

3. Создать экземпляр класса Admin:

admin = Admin(app)

4. Добавить представление к каждой модели:

admin.add_view(ModelView(User, db.session))
admin.add_view(ModelView(Orders, db.session))
admin.add_view(ModelView(Clients, db.session))

5. Перейти на /admin и посмотреть что получилось:

Вот код примера, чтобы сразу попробовать админку в действии:

from flask import Flask
from flask_sqlalchemy import SQLAlchemy
from flask_admin import Admin
from flask_admin.contrib.sqla import ModelView

app = Flask(__name__)
app.secret_key = "randomstring"
app.config['SQLALCHEMY_DATABASE_URI'] = "sqlite:///:memory:"
app.config['SQLALCHEMY_TRACK_MODIFICATIONS'] = True

db = SQLAlchemy(app)

class User(db.Model):
    __tablename__ = 'userz'
    uid = db.Column(db.Integer, primary_key=True, autoincrement=True)
    name = db.Column(db.String(50))
    email = db.Column(db.String(50))

class Order(db.Model):
   __tablename__ = 'orders'
   uid = db.Column(db.Integer, primary_key=True, autoincrement=True)
   description = db.Column(db.String(500))
   email = db.Column(db.String(50))

class Client(db.Model):
   __tablename__ = 'clients'
   uid = db.Column(db.Integer, primary_key=True, autoincrement=True)
   name = db.Column(db.String(50))
   email = db.Column(db.String(50))

db.create_all()

mock_users = []
mock_users.append(User(name="Alex", email="alex@gmail.com"))
mock_users.append(User(name="Darina", email="darina@gmail.com"))
mock_users.append(User(name="Prohor", email="prokhor@gmail.com"))
db.session.add_all(mock_users)
db.session.commit()

admin = Admin(app)
admin.add_view(ModelView(User, db.session))
admin.add_view(ModelView(Order, db.session))
admin.add_view(ModelView(Client, db.session))


app.run(debug=True)

Расширение ModelView

ModelView гибко настраивается. Мы можем управлять:

– Моделями, а также полями, которые можно будет редактировать;

– Организацией меню в админке;

– Тем, по каким полям будет доступен поиск и фильтр.

Flask-admin предлагает необычный способ настроить эти свойства.

1. Создать новое представление, унаследовав его от ModelView.

2. Задать в нем нужные свойства.

3. Добавить представление через admin.add_view()

class MyUserView(ModelView):# Настройка общего списка

  column_exclude_list = ['password']  # убрать из списка одно или несколько полей
  column_searchable_list = ['name', 'email']  # добавить поиск по полям
  column_filters = ['country'] # добавить фильтр по полям# Права на CRUD операции

  can_create = False
  can_edit = False
  can_delete = False


  # Лимит записей на страницу

  page_size = 50

...

admin.add_view(MyUserView(User, db.session))

Больше настроек в документации:https://flask-admin.readthedocs.io/en/latest/introduction/#customizing-built-in-views
