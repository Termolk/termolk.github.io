4.5 Объектно-Реляционное Отображение (ORM)

Язык программирования — это инструмент для обработки данных из одного состояния в другое. Если мы хотим сохранить состояние данных навсегда, для этого потребуется некоторое хранилище. Обычно для этой цели используют реляционную базу данных, но  возникает проблема: языки программирования и базы данных работают с данными по-разному. Кроме того, реляционные базы данных имеют свой язык именования SQL (Structured Query Language).

Если вы изучали SQL ранее - отлично! Если нет – посмотрите соответствующий курс на Stepik или попробуйте интуитивно догадаться, что значат строчки на SQL - они достаточно понятны, если вы знаете английский на базовом уровне.

SQL – норм, но это не точно

Например, если вы храните информацию о пользователях в базе данных, для изменения имени пользователя с номером 244, вам будет достаточно написать такое выражение на SQL:

UPDATE `users` SET `email` = "masha@stepik.org" WHERE `id` = "244"

Такая команда называется SQL запросом. У этого языка есть свой синтаксис, свои особенности и свои подводные камни. Однако, каждому разработчику проще и приятнее работать с данными на своем любимом языке, а в случае Python – по возможности использовать объекты. Да, код выше выглядит неплохо, но было бы лучше написать вместо него что-то вроде:

users.magic_get(244).email = 'masha@stepik.org'Другой пример. Если вы создаете нового пользователя в базе данных, операция на SQL будет выглядеть так

INSERT INTO users(name, email, phone) VALUES ('Alice','alice@stepik.org',"+134545445");

Но было бы здорово использовать Python и написать что-то подобное:

alice = Users("name"="Alice","email"="alice@stepik.org","phone"="+134545445")

Последний пример - удаление.

DELETE FROM `users` WHERE `id` = "244"

Было бы неплохо заменить на

Users.magic_delete(244)

Хорошая новость

К счастью, мы не первые люди на Земле, которые страдают от этой беды. Существует подход, называемый Объектно-Реляционное Отображение (Object-Relational Mapping, далее будет использоваться аббревиатура — ORM), который способен «переводить» запросы с «объектного» языка на язык «базы-данных» и обратно! Вы можете использовать набор классов, которые предоставляет ваш фреймворк для всего цикла CRUD, то есть для того, чтобы:

– Создавать записи в базе данных (create)

– Получать их из базы данных (read)

– Редактировать эти записи (update)

– Удалять их (delete)

Концепция Объектно-Реляционного отображения

Почти все языки программирования имеют библиотеки ORM, но прежде чем начать их использовать, давайте выясним, в чем заключается концепция ORM.

Объектно-реляционное отображение — это концепция преобразования данных из объектно-ориентированного языка программирования в представление реляционной базы данных и наоборот. Это решает проблему согласования двух систем разных типов и синхронизации потока данных между ними.

ORM включает в себя:

  • Виртуальные таблицы.
  • Отношения.
  • Операции с объектами.

Так что же это за составляющие и зачем они нам нужны? Давайте рассмотрим подробнее каждый пункт.

Виртуальные таблицы

Реляционные базы данных используют кортежи и таблицы для хранения данных. Большинство языков программирования имеют кортежи, но не имеют таблиц. Каким же образом можно представить их на языке программирования?

Основная идея заключается в использовании классов в качестве описания таблиц. Необходимо создать класс как виртуальную таблицу для данной таблицы в базе данных. Мы используем или определяем методы для этого класса чтобы извлечь, изменить и удалить данные.

Чтобы представить одну строку из таблицы, мы можем определить другой класс (для некоторых библиотек это может быть тот же класс) и сопоставить его атрибуты со столбцами таблицы. Экземпляр этого класса может манипулировать не только значениями строки, но и отношениями, которые имеет этот ряд. Далее обсудим это подробнее на примере.

На рисунке представлен пример с классом City. Мы сопоставляем значения в таблицах со скалярными атрибутами в классе. Имя (name) может быть строкой. Долгота (longitude) и широта (latitude) могут быть действительными числами.

Как известно, класс может иметь атрибуты, представляющие список других объектов. Например, в городе может быть много улиц. Это противоречит отображению атрибута в столбец, но ORM предоставляет инструменты и для этих случаев. В терминах реляционных баз данных связь между одним объектом и несколькими другими называется отношением «один ко многим». Это очень похоже на атрибут списка объекта.

Отношения

Отношение — это ссылка, которая связывает значение из одной таблицы со строкой в другой. База данных может хранить такие ссылки как ключи. Вы можете думать об этом как об объекте, содержащем другой объект как атрибут.

Отношения в базах данных — это не просто ссылки. Когда вы удаляете корневую строку из одной таблицы, это может подразумевать каскадное удаление всех связанных строк из других таблиц. Это не та же логика, что и для языка программирования. Если у нас есть ссылка в объекте, то ожидаем, что при удалении этой ссылки исчезнет только ссылка, а не сам связанный объект. С другой стороны, если в базе данных есть таблица «Город», а в ней, например, строка «Стамбул», то можно предположить, что при её удалении из таблицы все связанные строки из таблицы «Улица» также исчезнут. Улица принадлежит городу, а без города нет улиц.

Мы можем представить этот пример как класс City со списком улиц или в виде множества экземпляров класса Street с атрибутом City.

ВАЖНО! Вы можете ожидать, что при удалении города также удаляются все соответствующие ему улицы. С другой стороны, если вы удалите улицу с атрибутом city, город останется в базе данных. Это означает, что отношения имеют направления. Каскадные удаления подразумевают удаление зависимых объектов, не зависящих от них.

Операции с объектами

Четыре часто используемые операции со строками в базе данных называются операциями CRUD (создание, чтение, обновление, удаление). Это похоже на то, что можно делать с объектами на языке программирования.

После прочтения о виртуальных таблицах и отношениях вы можете попытаться управлять ими с помощью библиотеки ORM. Библиотеки обычно предоставляют команды высокого уровня, которые похожи на работу с любыми другими объектами в языке программирования. Знания об отношениях и таблицах помогут не повредить данные в базе данных.

Настоятельно рекомендуем прочитать документацию для библиотеки ORM перед ее использованием. Это поможет познакомиться с последствиями и значениями прикладных операций.

Плюсы и минусы

Рассмотрим плюсы и минусы ORM, для того, чтобы вы могли определиться использовать или нет данную концепцию для своего проекта.

Плюсы:

  • Вы работаете с базой данных так же, как вы пишете другой код;
  • Библиотека изолирует ваш код от специфики языка SQL базы данных, которую вы используете;
  • Код, использующий ORM, обычно легче читать, писать и поддерживать;
  • Вы не должны знать SQL для работы с базой данных.

Минусы:

  • Иногда библиотеки ORM генерируют неэффективные запросы к базе данных;
  • Сложно контролировать генерацию запросов;
  • Вы не можете использовать все возможности и преимущества конкретной базы данных, которая у вас есть;
  • Вы должны научиться работать с библиотекой.

В то время как ваш проект предназначен для сотен или даже тысяч пользователей, библиотеки ORM могут быть хорошим выбором. Если же вы планируете создать новый Facebook или Google, возможно, ORM — это не то, что вам нужно для такой амбициозной цели!

Чтобы начать работать с библиотекой ORM, вы можете рассмотреть SQLAlchemy для Python.
