4.3 Что такое SQL

Введение

SQL (Structured Query Language/Язык Структурированных Запросов) является предметно-ориентированным языком программирования, предназначенным для обработки данных в таблицах. Он был разработан в 1970-х годах, и по сей день SQL-подобные интерфейсы широко используются и поддерживаются в системах управления данными, даже тех, которые не основаны на концепции таблицы.

Из-за возможности его крайне разностороннего применения вы, вероятно, найдете SQL весьма полезным. Если вы инженер по программному обеспечению, полезно изучить его, поскольку многие программные системы хранят и обрабатывают бизнес-данные с помощью служб, поддерживающих SQL. Например, информационная система страховой компании может использовать SQL для извлечения и обновления данных о клиентах.

Если вы заинтересованы в аналитике, с помощью SQL вы сможете легко объединять данные для статистических вычислений. Предположим, вам необходимо оценить изменения популярности имени Jessie между 1920 и 2000 годами (включительно) на основе данных переписи. В SQL эта задача займет всего 11 строк кода! Вы можете не понимать код сейчас, просто попробуйте прочитать его как предложение на английском языке. Он отбирает записи о лицах с именем "Jessie ", родившихся между 1920 и 2000 годами; групповые записи для каждой пары года и пола; подсчитывает количество записей в каждой группе; и в результате генерирует таблицу со столбцами «year», «gender» и «cnt», отсортированными по году и полу.

Если вы работаете в компании, работающей с данными, то знайте, что SQL - это стандарт языка манипулирования данными.

Из SQL можно многое почерпнуть и многому научиться. Мы предлагаем начать с самых основ: разобраться, что означает SQL. Давайте разберем аббревиатуру, чтобы знать, с чем мы имеем дело.

S значит Структурированный

SQL - это язык, используемый для извлечения и обновления данных, структурированных в виде таблиц. Такие данные появляются в различных прикладных областях: например, в листах Excel с бухгалтерией, статистическими данными переписи в Google BigQuery или в онлайн-магазинах, где используется специальная система программного обеспечения для хранения и использования таблиц. Называется она системой управления реляционными базами данных (RDBMS) и она помогает обрабатывать информацию о товарах, заказах и клиентах.

SQL предназначен для таблиц со следующей структурой: таблица содержит строки, каждая из которых представляет объект, и столбцы с именами атрибутов этих объектов. Ячейка таблицы из строки R и столбца C хранит значение атрибута C объекта R. Например, в таблице «census» из нашего примера строки представляют людей, и каждый человек имеет атрибуты «id», «year»,  «name» и  «gender». Например, третий ряд содержит данные о Вилли, который родился в 1985 году.

Довольно часто данные объединяются в несколько таблиц с именами, обычно это называют база данных, и к этим таблицам можно обращаться по их именам. Например, в базе данных для интернет-магазина таблица «Customers» содержит данные о клиентах (их имена и контакты), а «Orders» хранит информацию о заказах (клиент, товары, реквизиты платежа).

Q значит Запрос

SQL - это язык программирования с большим набором функций для обработки данных. SQL является декларативным языком, и, таким образом, любой оператор, написанный на нем, является запросом, который сообщает системе, что должно быть сделано или оценено, но не как должно быть.

Давайте сосредоточимся на самой простой и основной функциональности извлечения данных из таблицы. Например, если у нас есть таблица с именем «Customers», запрос, извлекающий все строки и столбцы, выглядит следующим образом:

SELECT * FROM Customers;

L значит Язык

Этот простой запрос из приведенного выше примера можно прочитать как "select everything from customers". SQL специально был разработан  похожим на естественный язык для нашего с вами удобства. Декларативная природа SQL скрывает сложности от пользователя и, в некоторой степени, вы просто декларируете свою волю, пока система анализирует запрос, выбирает поток управления и выполняет его.

Так же, как и любой естественный язык, SQL имеет стандарт Американского национального института стандартов (ANSI) и он реализован поставщиками программного обеспечения во многих диалектах, которые поддерживают SQL. Как правило, диалекты основаны на стандарте, но они до сих пор показывают различия в некоторых технических деталях, например, при обработке даты и строки. Это означает, что запросы SQL на разных диалектах несовместимы. Однако, как только вы освоите основы SQL, вы сможете адаптироваться к различным диалектам, например, к американскому и британскому английскому.

Здесь и далее, в случае несовместимости между поставщиками и стандартом ANSI, мы предоставляем синтаксис для PostgreSQL.

Заключение

Теперь вы знаете, что SQL является предметно-ориентированным декларативным языком и предназначен для тех, кто работает со структурированными данными.

Если у вас есть данные, которые можно организовать в виде таблиц, и вы хотите знать, как выбирать строки и столбцы в соответствии с некоторыми критериями, объединять факты, создавать группы объектов, вычислять статистику и многое другое, давайте углубимся в SQL.
