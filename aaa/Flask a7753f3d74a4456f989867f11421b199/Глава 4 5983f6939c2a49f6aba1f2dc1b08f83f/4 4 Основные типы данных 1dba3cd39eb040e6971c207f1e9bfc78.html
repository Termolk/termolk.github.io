4.4 Основные типы данных

Введение

Как вы уже знаете, SQL - это язык для работы с данными различных типов, организованными в таблицу.

Обычно, значения данных из одного столбца в таблице имеют одинаковый тип.Например, таблица Car может выглядеть так:

Мы видим, что значения в столбце года изготовления (manufacture year) представляют собой целые числа, значения цены (price) - десятичные, а значения электрический (electricity) - логические. Базы данных SQL обычно требуют, чтобы у каждого столбца в таблице базы данных было имя и тип данных. Тип данных столбца ограничивает набор значений, которые могут храниться в столбце, и определяет набор возможных операций с ними.

Стандарт ANSI определяет довольно сложный набор типов данных. Кроме того, поставщики баз данных обычно добавляют свои нестандартные параметры данных.

В этой теме мы рассмотрим очень простое подмножество типов данных: INTEGER, REAL, DECIMAL, VARCHAR, и BOOLEAN.

Integer

INTEGER - это числовой тип данных, представляющий некоторый диапазон математических чисел. Обычно системы управления базами данных поддерживают диапазон от -2147483648 до +2147483647, что достаточно для большинства бизнес-задач.

Тип INTEGER подходит для счетчиков, числовых идентификаторов и любых целочисленных бизнес-значений, которые можно себе представить и которые соответствуют диапазону шкалы. Например, в базе данных рекламной кампании может быть таблица, в которой хранится идентификатор баннера, количество кликов по нему и количество показов баннера: все в столбцах INTEGER.

Real

Числовые данные, конечно, выходят за пределы целочисленных значений: например, рассмотрим расстояние от Земли до Марса (225,0e+06 км), количество белых клеток в крови (от 1,2e+09 до 3,0e+09 на литр) или масса электрона (9.10938356e-31 кг). Для таких реальных значений SQL имеет специальный тип данных с интуитивно понятным именем REAL. Этот тип данных часто называют FLOAT, отражая способ внутреннего хранения чисел: экспоненциальная форма используется и, таким образом, десятичная точка может «плавать», то есть она может быть размещена в любом месте относительно значащих цифр числа.

Этот числовой тип обычно называется неточным, поскольку некоторые значения хранятся в виде приближений (например, 0,1), поэтому сохранение и получение значения могут не всегда работать так, как ожидается. Таким образом, если требуется точное хранение и расчеты (например, для биллинговой базы данных), REAL не ваш выбор. Однако РЕАЛЬНЫЕ данные часто встречаются в системах, которые работают с очень маленькими и очень большими действительными числами и требуют быстрого времени обработки. Например, расстояние между галактиками или диаметр атомного ядра может храниться в одном столбце типа REAL. В большинстве систем тип REAL имеет диапазон шкалы от 1e-37 до 1e+37 с точностью не менее 6 десятичных цифр; абсолютная точность для малых чисел выше, чем для больших.

Decimal

В повседневной жизни мы обычно сталкиваемся с десятичными числами довольно часто: например, при измерении температуры тела (36,6 градусов Цельсия) или веса (52,700 кг), оценке высоты знаменитой Пизанской башни (55,86 м) и, конечно же, считая наши драгоценные финансы ($ 103050,79). SQL поддерживает специальный тип данных для таких значений – DECIMAL (precision, scale).

Как видите, этот тип имеет два параметра: precision (точность) and scale (масштаб). Scale определяет количество цифр в дробной части, справа от десятичной точки. Precision - это общее количество цифр в числе, то есть по обе стороны от десятичной точки.

Тип DECIMAL обычно называют точным числовым типом с неограниченной точностью и масштабом; однако есть предел - до 131072 цифр перед десятичной запятой и до 16383 цифр после десятичной запятой. Если подумать, этого должно быть достаточно!

Одним из наиболее важных различий между типами DECIMAL и REAL для приложений являются правила округления. ДЕСЯТИЧНЫЕ значения округляются интуитивно и предсказуемо, например, округление -0,5 приведет к -1. Правила округления для типа REAL могут быть менее интуитивными, и здесь округление -0,5 даст нам -0. Однако, арифметические операции над типами REAL выполняются быстрее, а сами значения занимают меньше места для хранения.

Text

Конечно, может потребоваться обработать что-то, кроме числовых данных, и SQL поддерживает семейство типов данных, предназначенных для представления текстовых данных. Давайте рассмотрим один из них, достаточно универсальный и базовый – VARCHAR(n).

Этот тип представляет собой строку символов различной длины, не длиннее n. Например, можно вставить строки «apple», «plum» и «peach» в столбец с типом VARCHAR(5). Строки «orange» или «banana» не будут соответствовать ограничению по длине, и система либо урезает их, либо выдаст ошибку, если попытается вставить такие длинные значения.

Boolean

BOOLEAN тип представляет значения логической (истинной) логики: TRUE или FALSE. Этот простой тип данных может использоваться для любых атрибутов с семантикой флагов, например, посещал ли клиент сайт конкурента.

Кто определяет типы и как?

Как пользователь базы данных, вы должны просто знать типы столбцов таблицы, которые используете, чтобы иметь возможность обрабатывать их правильно. Однако, как программист, вы должны знать, как создать таблицу и определить типы столбцов.

Давайте рассмотрим пример запроса SQL, который определяет таблицу «Census» с 5 столбцами:"id" с типом INTEGER, "name" с типом VARCHAR(20), "birth_place_latitude" с типом REAL, "year_income" с типом DECIMAL(20,3), and "is_male" с типом BOOLEAN.

CREATE TABLE Census (
    id INTEGER,
    name VARCHAR(20),
    birth_place_latitude REAL,
    year_income DECIMAL(20,3),
    is_male BOOLEAN
);

Можно увидеть, что шаблон выглядит следующим образом:

CREATE TABLE table_name (column_name_1 column_type_1,..., column_name_n column_type_n);

Заключение

Данные могут быть очень разнообразными, и SQL поддерживает обширный набор типов данных для представления этого разнообразия. Мы обсудили базовое подмножество типов данных, но существуют также: приведение типов, составные типы, специальные типы для числовых, текстовых и временных меток и так далее.
