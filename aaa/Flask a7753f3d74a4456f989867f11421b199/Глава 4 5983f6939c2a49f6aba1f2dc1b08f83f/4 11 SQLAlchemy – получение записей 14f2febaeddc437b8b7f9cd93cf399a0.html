4.11 SQLAlchemy – получение записей

Почти за каждой страничкой сайта, которую вы открываете стоят запросы к БД.

Вы решили узнать, что идет в кино и зашли на сайт киноафиш? Сервер сайта извлекает список фильмов в категории "новинки".

Хотите узнать больше про фильм? Сервер снова обращается к таблице "фильмы", но уже, чтобы получить описание, бюджет, сборы, рейтинги и всякую другую полезную информацию. Хотите посмотреть, где идет вот этот новый фильм про супергероев? Сервер делает выборку из таблицы "сеансы". Хотите почитать, кто играет в этом фильме? Сервер вынимает актеров из таблицы "актеры", которые связаны с этим фильмом.

Пора и нам научиться извлекать данные из созданных таблиц!

В этом топике вы научитесь:

  • Получать несколько записей из базы данных
  • Получать запись по id
  • Получать записи в определённом порядке

Получение набора записей

В SQL получение записей из таблицы называется выборкой, и делается с помощью запроса начинающегося с ключевого слова SELECT. В ответ на запрос база возвращает нам набор записей. В CRUD это буква R, read. Удобство декларативного SQL в том что нам не нужно знать как база хранит данные физически, достаточно указывать что именно мы хотим получить.

В SQLAlchemy за получение объектов из базы ответственен объект Query, в нашем случае db.session.query .

Когда мы хотим добавить дополнительные критерии, мы не изменяем уже созданный объект Query, а создаём его копию с измененным содержимым. В Python таким же образом ведут себя строки, мы не можем изменить строку напрямую, а все методы строки, которые что-либо в ней меняют, на самом деле возвращают новую строку с соответствующими изменениями.

Для начала получим всех наших пользователей. Для этого укажем, какую модель будем использовать, передав её аргументом.

users = db.session.query(User).all()

Мы получили объект класса Query, который будет выбирать записи из таблицы, на которую указывает модель User. После у нашего объекта мы можем вызывать различные методы, которые будут создавать дополненный условиями объект Query. Или же мы можем вызвать метод Query.all, который указывает SQLAlchemy, что мы хотим получить все объекты заданной Query себе.

В момент вызова .all будет сгенерирован запрос типа:

SELECT users.id AS users_id, users.name AS users_name, users.email AS users_email, users.password AS users_password
FROM users

Как мы видим SQLAlchemy берёт информацию о названии полей и таблицы из модели, и указывает их в запросе.

Полный код примера:

class User(db.Model):
    __tablename__ = "users"

    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String, nullable=False)
    email = db.Column(db.String, nullable=False)
    password = db.Column(db.String, nullable=False)


# Следующие строчки добавляют записи в базу
db.session.add(User(name="Марк", email="mark@stepik.org", password="himark111"))
db.session.add(User(name="Лиза", email="lisa@stepik.org", password="whatamidoing222"))
db.session.commit()

# Теперь выведем записи из базы
users = db.session.query(User).all()
print(users)

Работа с результатами запроса

После того как мы написали запрос, обычно мы хотим получить записи из базы. Для этого у объекта Query есть несколько методов которые возвращают результаты запроса.

Это Query.all, Query.scalar и Query.get.

Порядок

Также мы можем указывать порядок, в котором база будет возвращать записи.

Важно — если порядок не указан явно, БД имеет полное право вернуть записи в любом порядке, и он не обязан даже совпадать от запроса к запросу.

Для этого нам понадобится отсортировать записи в ответе. В объекте Query для этого есть метод .order_by. Он создаст новый объект Query, в котором будет указано как сортировать и база сама отсортирует нам записи. Создадим нужный нам объект Query:

class Product(db.Model):
    __tablename__ = "products"

    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String, nullable=False)
    created_at = db.Column(db.DateTime, nullable=False)


products_query = db.session.query(Product).order_by(Product.created_at).limit(10)

И получим все данные по нашему запросу. Заодно убедимся, что получили столько записей сколько мы хотели, и не больше:

products = products_query.all()
print("Получили", len(products), "продуктов")

Заодно давайте выведем имя и дату создания продуктов, чтобы точно убедиться, что они идут по порядку и от самых старых к самым новым:

for product in products:

Лимит на количество записей

Обычно мы хотим получить себе в ответ не всю базу данных сразу, а только часть содержащихся в ней данных. В Query объекте для этого есть метод .limit(n). Он создают новый объект Query, в котором будет указано сколько записей мы хотим получить.

class Product(db.Model):
    __tablename__ = "products"

    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String, nullable=False)
    created_at = db.Column(db.DateTime, nullable=False)


products_query = db.session.query(Product).order_by(Product.name.desc()).limit(3)
products = products_query.all()
print("Получили", len(products), "продуктов")
for product in products:
    print("Продукт", product.name, "был создан", product.created_at)

Получение одной записи

Что если мы хотим получить строго один объект? Типичная задача, пользователь делает запрос, в котором указывает свой id, и мы хотим получить объект пользователя с этим id из базы.

Для этого случая у query есть метод get. Он принимает в качестве аргумента первичный ключ и возвращает строго одну запись, так как в таблице не могут содержаться несколько записей с одинаковым первичным ключом.

Например, мы знаем что у нас есть пользователь с id равным 1.

Тогда мы можем использовать user = db.session.query(User).get(1) ,  чтобы получить одну запись по ее id.

Полный код примера:

from flask import Flask
from flask_sqlalchemy import SQLAlchemy

app = Flask(__name__)
app.secret_key = 'randomstring'
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///products.db'
app.config['SQLALCHEMY_TRACK_MODIFICATIONS'] = False
db = SQLAlchemy(app)

class User(db.Model):
    __tablename__ = "users"

    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String, nullable=False)
    email = db.Column(db.String, nullable=False)
    password = db.Column(db.String, nullable=False)

db.create_all()

_users = []
_users.append(User(name="Артимович Лева Алексеевич", email="Ohota257@gmail.com", password="sAtMnZgiB9d2"))
_users.append(User(name="Фёдорова Таммия Георгиевна", email="FedorovaTammiya32@gmail.com", password="96r1C7TbRJmo"))
_users.append(User(name="Крылов Васька Макарович", email="KryilovVaska145@gmail.com", password="sLhczSy3YUWg"))

db.session.add_all(_users)
db.session.commit()

user = db.session.query(User).get(1)
print(user.id, user.name)

Такой код вернет нам:

1 Артимович Лева Алексеевич

Несуществующий ключ

Если в таблице нет записи с нужным id, get вернёт None.

user = db.session.query(User).get(-1)
print('Is user None?', user is None)

В свою очередь, в вебе часто нужно, если объекта с нужным id в базе не нашлось, вернуть пользователю ответ с кодом 404 Not found. Для этой задачи Flask-SQLAlchemy добавляет в объект Query метод .get_or_404, который как можно угадать по названию или возвращает нам объект по id, или возвращает пользователю ответ со статус кодом 404.

Например, мы хотим вернуть имя пользователя по его id или 404 если такого пользователя нет в базе.

Полный код примера:

from flask import Flask
from flask_sqlalchemy import SQLAlchemy


app = Flask(__name__)
app.config["SQLALCHEMY_DATABASE_URI"] = "sqlite:///users.db"app.config['SQLALCHEMY_TRACK_MODIFICATIONS'] = False
db = SQLAlchemy(app)


class User(db.Model):
    __tablename__ = "users"

    id = db.Column(db.Integer, primary_key=True)
    name = db.Column(db.String, nullable=False)
    email = db.Column(db.String, nullable=False, unique=True)
    password = db.Column(db.String, nullable=False)

db.create_all()

_users = []
_users.append(User(name="Артимович Лева Алексеевич", email="Ohota257@gmail.com", password="sAtMnZgiB9d2"))
_users.append(User(name="Фёдорова Таммия Георгиевна", email="FedorovaTammiya32@gmail.com", password="96r1C7TbRJmo"))
_users.append(User(name="Крылов Васька Макарович", email="KryilovVaska145@gmail.com", password="sLhczSy3YUWg"))

db.session.add_all(_users)
db.session.commit()

@app.route('/user/<id>')
def show_name(id):
    user = db.session.query(User).get_or_404(id)
    return user.name

app.run()

Запустите этот код, затем перейдите на id существующего и несуществующего пользователя, чтобы получить ошибку.

Альтернативный короткий синтаксис

Все перечисленные раньше операции можно писать в сокращенном виде:

Получение всех записей:

User.query.all()

Получение одной записи:

User.query.get(1)

Упорядочивание:

User.query.order_by(User.mail).all()

Лимит на количество записей:

User.query.limit(5).all()

Обратите внимание: такой синтаксис, в отличие от db.session... работает только во flask-sqlalchemy и не работает в sqlalchemy самостоятельной.

Резюме

db.session.query(Модель).all() возвращает все записи

users = db.session.query(User).all()

User.query.all()

db.session.query(Модель).get(id) возвращает одну запись по ее id.

user = db.session.query(User).get(1)

.order_by определяет параметр сортировки

db.session.query(Product).order_by(Product.created_at)

.desc инвертирует сортировку

db.session.query(Product).order_by(Product.created_at.desc())

.limit задает количество, которым будет ограничен результат

db.session.query(Product).limit(5)

Получение всех записей:

User.query.all()

Получение одной записи:

User.query.get(1)

Упорядочивание:

User.query.order_by(User.mail).all()

Лимит на количество записей:

User.query.limit(5).all()
